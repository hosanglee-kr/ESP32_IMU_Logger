# 📚 SMEA-100 핵심 아키텍처 및 DSP 문제/개선 요약 보고서 (버전: v009_005 기준)
본 문서는 T440_FeatExtra (특징량 추출) 및 T430_DspEng (신호 정제) 모듈의 실시간 100Hz 처리 한계와 부동소수점(Float32) 정밀도 문제를 극복하기 위해 식별된 문제점과 해결책을 정리합니다.
## 1. T440_FeatExtra (특징량 추출) 모듈
### 🚨 문제점 1: 스펙트럼 누출로 인한 N-Top Peak 중복 검출 (Side-Lobe 문제)
 * **현상:** FFT의 윈도윙 과정에서 발생하는 에너지 누출로 인해, 진짜 결함 주파수 옆에 붙은 가짜 피크(어깨)들이 Top 5 피크를 모두 채워버리는 현상.
 * **해결 방안:** **NMS (Non-Maximum Suppression) 도입.** * 설정된 진폭 하한선(peak_amplitude_limit_min) 이하의 피크를 일차적으로 쳐냄.
   * 가장 강한 피크부터 선택한 뒤, 설정된 주파수 간격(peak_freq_gap_limit_hz_min) 이내에 있는 하위 피크들을 강제 탈락시켜 독립적인 결함 주파수만 도출.
### 🚨 문제점 2: 켑스트럼(Cepstrum) 타겟 탐색 시 수학적/논리적 오류
 * **현상:** 1. Quefrency 인덱스를 구할 때 단위 변환 오류로 인해 배열을 크게 이탈하는 버그.
   2. 탐색 구간에 0번 인덱스(DC 성분)가 포함되어 무조건 0번이 최대값으로 잡히는 맹점.
   3. FFT 결과에서 실수부(Real)만 추출하여 복소수 크기(Magnitude)가 왜곡됨.
 * **해결 방안:**
   * 주파수(Hz)를 Quefrency(sec)로 역산한 뒤 샘플링 레이트를 곱해 정확한 인덱스 매핑 수행.
   * 탐색 시작 인덱스를 max(1, idx)로 강제하여 DC 성분 완전 배제.
   * sqrtf(real^2 + imag^2) 수식으로 완벽한 복소수 크기 연산 수행.
### 🚨 문제점 3: 공간 위상(Spatial Phase) 분석 시 윈도우 미적용의 맹점
 * **현상:** 위상을 보존하겠다는 명목으로 Hanning Window를 뺀 채 FFT를 돌려 심각한 스펙트럼 누출이 발생, 위상 데이터 자체가 쓰레기 값(Garbage)으로 오염됨.
 * **해결 방안:** L/R 두 채널 모두에 **동일한 Hanning Window를 곱하여** 누출을 막으면서도 두 채널 간의 "상대적 위상차(Phase Difference)"는 완벽하게 보존하는 방향으로 수학적 오류 교정.
### 🚨 문제점 4: STA/LTA 비율 연산 시 '슬라이딩 윈도우' 누락
 * **현상:** 1프레임(1024 샘플)의 가장 앞부분 1ms, 10ms만 비교하여, 프레임 중간이나 끝에서 터지는 돌발 충격음(Impulse)을 감지하지 못하는 사각지대 존재.
 * **해결 방안:** 프레임 전체를 순회하며 STA/LTA 비율의 **Max 값을 스캔하는 '슬라이딩 윈도우(Sliding Window)'** 방식으로 연산 로직 변경 필요.
### 🚨 문제점 5: 무게 중심(Spectral Centroid) 계산 시 DC 바이어스
 * **현상:** 0Hz(DC 성분)에 쌓인 막대한 에너지 때문에 무게 중심이 0Hz 쪽으로 강제로 끌려내려가 소리의 특성을 잃어버리는 현상.
 * **해결 방안:** 주파수 누적 합산 루프를 0이 아닌 **1 (첫 번째 AC 성분 빈)부터 시작**하여 DC 바이어스를 원천 차단.
### 🚨 문제점 6: Band RMS 계산 시 총합 vs 평균의 모순
 * **현상:** 설정된 주파수 밴드 안의 에너지를 구할 때 빈(Bin)의 개수로 나누어 '평균'을 구함. 밴드폭이 넓어지면 전체 에너지가 큰데도 수치가 작아지는 왜곡 발생.
 * **해결 방안:** 빈의 개수로 나누는 로직을 제거하고, 밴드 내 파워를 **'총합(Total Energy)'** 형태로 구하여 밴드폭에 구애받지 않고 절대 에너지를 비교하도록 수정.
## 2. T430_DspEng (신호 정제 엔진) 모듈
### 🚨 문제점 1: LTI 법칙 무시로 인한 연산량 50% 낭비 (Beamforming 위치)
 * **현상:** 가장 무거운 FIR 필터 등을 L/R 두 채널에 각각 두 번씩 돌리고 난 뒤에 빔포밍(병합)을 수행함.
 * **해결 방안:** 필터의 선형성(LTI)을 활용하여 **빔포밍(L+R)을 파이프라인 가장 맨 앞으로 이동.** 1채널로 만든 뒤 필터링을 한 번만 돌려 CPU 부담 절반 감소.
### 🚨 문제점 2: 노이즈 게이트(Noise Gate)로 인한 인위적 스파이크 생성
 * **현상:** 노이즈 게이트로 파형을 0으로 꺾어버린(계단파) 직후, 프리엠파시스(미분기)를 통과시켜 무한대 스파이크(Dirac Delta)가 튀어 오르는 치명적 오류.
 * **해결 방안:** 순서를 뒤집어 **프리엠파시스로 파형을 부드럽게 강조한 뒤, 맨 마지막에 노이즈 게이트를 배치**하여 잔여 노이즈를 뮤트(Mute) 처리.
### 🚨 문제점 3: DC Removal 연산의 낭비
 * **현상:** FIR HPF가 0Hz 성분을 완벽히 깎아내는데도 불구하고, 배열 전체를 순회하며 평균값을 빼는 DC Removal 작업을 불필요하게 수행 중.
 * **해결 방안:** **FIR HPF가 켜져 있으면 DC Removal 로직을 완전히 패스(Bypass)**하도록 제어하여 사이클 확보.
### 🚨 문제점 4: 필터 체인(Filter Chain) 배치 오류 (Ringing 유발)
 * **현상:** FIR 필터 이후에 IIR Notch 필터를 배치하여 위상 왜곡 및 스파이크성 링잉 현상 발생 위험 증가.
 * **해결 방안:** **Median -> IIR Notch -> FIR LPF/HPF** 순서로 전진 배치하여 전기적/스파이크성 노이즈를 1차적으로 억제한 후 FIR 필터로 선형 위상을 유지하며 대역폭을 깎아내는 정석적 필터 체인으로 구조 개편.
## 3. 공통 방어 (안정성 및 메모리)
 1. **PSRAM 힙 강제 할당:** FFT_SIZE를 넘어서는 거대 텐서나 행렬(_fftWorkBuf, _melBankFlat 등)은 태스크 스택 오버플로우를 막기 위해 반드시 init() 단계에서 MALLOC_CAP_SPIRAM을 통해 16바이트 정렬로 힙 영역에 할당.
 2. **SIMD NaN 독성(Poisoning) 차단:** 행렬 곱셈 전, 배열의 패딩 영역(사용되지 않는 쓰레기 값 영역)을 0.0f로 철저히 초기화(memset)하여 전체 텐서가 NaN으로 오염되는 현상 차단.
 3. **부동소수점 오버/언더플로우 방어:** 루트(sqrtf), 로그(logf), 나눗셈 연산 전 분모나 인자 값에 항상 fmaxf(val, EPSILON)을 씌워 NaN이나 Infinity로 뻗어버리는 하드웨어 크래시 원천 차단.
