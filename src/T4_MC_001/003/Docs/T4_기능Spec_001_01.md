SMEA-100(Super Micro Edge AI) 프로젝트의 성공적인 이식과 고도화를 위해, 샘플링 레이트 **42kHz** 상향 조정, **39차원 MFCC** 도입, 그리고 **ESP32-S3 N16R8** 하드웨어 사양을 반영한 **[최종 완결판 기능 명세서]**를 정리해 드립니다.
## 1. 하드웨어 상세 사양 (Hardware Specification)
### A. 메인 컨트롤러: ESP32-S3-WROOM-1-N16R8
 * **CPU:** Xtensa® Dual-Core 32-bit LX7 (최대 240MHz).
 * **AI 가속:** **Xtensa PIO(SIMD) 지침 세트** 탑재로 복소수 연산 및 FFT 가속 지원 (esp-dsp 라이브러리 최적화).
 * **Flash (N16):** **16MB Quad SPI Flash**. 대용량 전처리 로직 및 향후 ML 모델 탑재에 충분한 공간.
 * **PSRAM (R8):** **8MB Octal SPI PSRAM**. 42kHz 2채널 데이터를 실시간 버퍼링하고 ANC 및 FFT 연산을 수행하기 위한 필수 자원.
 * **I/O:** SDMMC(고속 SD카드), I2S(마이크 입력), GPIO(DC 트리거 입력).
### B. 센서: ICS43434 MEMS 마이크 (2개 조합)
 * **인터페이스:** I2S (Digital 24-bit).
 * **SNR:** 64 dB (고정밀 소음 분석에 적합).
 * **주파수 응답:** 50Hz ~ 20kHz (42kHz 샘플링 시 가청 대역 전체 커버).
 * **배치:** Broadside Array (20mm ~ 40mm 간격).
## 2. 통합 요구 사항 및 기능 Spec (Full Version)
### [Phase 1] 신호 취득 및 수집 제어
 1. **샘플링 레이트:** **42,000 Hz** (고주파 마찰음 및 고차 하모닉스 정밀 분석).
 2. **트리거 시퀀스:**
   * DC 4V 이상 감지 시 즉시 수집 시작 (3회 반복).
   * **1회차:** 시스템 워밍업 및 배경 노이즈 캡처.
   * **2, 3회차:** 실제 분석 대상 (교차 검증).
 3. **구간 자동 분할:**
   * **Noise Profile:** 시작 신호 후 0~150ms (ANC용 배경 소음 프로파일링).
   * **Active Signal:** 에너지 기반 자동 검출(STA/LTA)을 통한 실제 작동 구간 슬라이싱.
### [Phase 2] DSP 및 신호 전처리 파이프라인
 1. **개별 채널 처리:**
   * **DC 제거:** 하드웨어 편향 제거.
   * **Pre-Emphasis:** y[n] = x[n] - 0.97 \times x[n-1] (고주파 성분 복원).
   * **Notch Filter:** 60Hz/120Hz 및 주요 전원 하모닉스 제거 (IIR Biquad).
 2. **조합 및 정제:**
   * **Broadside Beamforming:** y = 0.5 \times (L+R) (정면 지향성 확보).
   * **FIR Band-pass:** 100Hz ~ 20kHz 대역 정제.
   * **ANC (Spectral Gating):** 노이즈 구간의 FFT 성분을 기반으로 실시간 소음 감쇄.
### [Phase 3] 고도화 피처 추출 (Feature Engineering)
 1. **시간 영역:** RMS, Energy, **Pooling StdDev(Min)**, **STA/LTA Count**, Crest Factor.
 2. **주파수 영역:** Spectral Centroid, Band RMS, **N-top Peaks(주파수/진폭)**.
 3. **고급 분석:**
   * **39차원 MFCC:** 13차 Coefficients + 13차 Delta + 13차 Delta-Delta.
   * **Cepstrum:** N1~N4 타겟 윈도우 기반 Max/MXRMS 추출.
 4. **동적/공간 특징:**
   * **Phase Features:** 위상차(IPD), Coherence (두 마이크 일관성).
   * **Delta Series:** 주요 피처의 시간적 변화율 측정.
### [Phase 4] 하이브리드 판정 및 저장
 1. **우선순위 판정 로직:**
   * **Level 1 (Test NG):** 트리거 미발생 또는 신호 누락.
   * **Level 2 (Rule NG):** 에너지 임계치 초과 또는 파형 표준편차 이상 (물리적 불량).
   * **Level 3 (ML Prediction):** MFCC 및 피처 기반의 딥러닝 예측 확률값 적용.
 2. **데이터 저장:** SD카드에 최종 정제된 **Raw PCM** 및 모든 피처를 담은 **JSON** 기록.
## 3. MFCC 및 Delta 구현 가이드 (42kHz 기준)
42kHz 환경에서 기계 소음의 특성을 가장 잘 포착하기 위한 설정값입니다.
### A. 윈도우 설정 (Windowing)
 * **Window Size (N_FFT):** **25ms** 권장. (42,000 * 0.025 = **1,050 samples**).
   * 연산 효율을 위해 **1,024 samples** (약 24.4ms)를 사용하고 Hann 윈도우를 적용합니다.
 * **Hop Length:** **10ms** (420 samples). 60% 정도의 오버랩을 유지하여 동적 변화를 놓치지 않도록 설계합니다.
### B. MFCC 구성 (39차원)
 1. **Static (13차):** 0~4,000Hz 대역에 26개의 Mel-filterbank를 배치하여 저주파(험)와 중주파(마찰음)를 조밀하게 분석 후 DCT를 통해 13개 추출.
 2. **Delta (13차):** 현재 프레임 전후 2프레임(N=2)을 참조하여 계산.
   * 수식: D_t = \frac{\sum_{n=1}^N n \times (C_{t+n} - C_{t-n})}{2 \times \sum_{n=1}^N n^2}
 3. **Delta-Delta (13차):** 위에서 구한 Delta 값의 변화량을 동일한 방식으로 한 번 더 계산.
## 4. 현재 PC 버전 vs. ESP32-S3 고도화 버전 비교
| 항목 | 현재 버전 (Windows PC) | 고도화 버전 (SMEA-100 / ESP32-S3) | 차이점 및 고도화 내용 |
|---|---|---|---|
| **처리 주체** | 고성능 CPU (Python 기반) | **ESP32-S3 N16R8 (C++ Native)** | 에지 단에서 즉각 처리, PC 의존성 제거 |
| **마이크 수** | 1채널 | **2채널 어레이 (Broadside)** | **공간 필터링(빔포밍)**으로 주변 소음 원천 차단 |
| **샘플링** | 42,000 Hz | **42,000 Hz (동일 유지)** | 고주파수 해상도를 그대로 유지하며 에지 이식 |
| **노이즈 제거** | 소프트웨어 기반 후처리 | **실시간 하이브리드 필터링** | **Notch + Beamforming + ANC** 3단계 정제 |
| **피처 세트** | Cepstrum, Band RMS 등 | **39D MFCC + Phase + Delta** | 위상 정보와 변화율을 추가하여 ML 판정력 극대화 |
| **트리거링** | 파일 기반 또는 전압 캡처 | **하드웨어 인터럽트 제어** | 시작/종료 신호와의 동기화 정밀도 향상 |
| **분석 구간** | 고정 구간 추출 | **에너지 기반 자동 구간 인식** | 제품 가동 시점의 미세 변화에 유연하게 대응 |
| **저장 방식** | HDF5 (대용량) | **PCM + JSON (분석 최적화)** | 현장 데이터의 가독성 및 학습용 데이터 가공 용이성 |
이 설계는 기존의 안정적인 판정 로직을 계승하면서도, **ESP32-S3의 하드웨어 가속**과 **2채널 위상 분석**이라는 에지 AI만의 강점을 극대화한 결과물입니다. 

이제 이 스펙을 바탕으로 **esp-dsp 기반의 MFCC 연산 루틴**이나 **42kHz용 IIR 필터 계수 설계**를 바로 진행하실 수 있습니다. 추가로 필요하신 부분이 있다면 말씀해 주세요!



SMEA-100 프로젝트의 성공적인 이식과 고도화를 위해, 제공해주신 과거 프로젝트(T20)의 뼈아픈 실수와 교훈을 바탕으로 **ESP32-S3 N16R8 환경에 최적화된 구현 원칙 및 주의 사항**을 재정리해 드립니다.
이 가이드는 **객체지향 설계(OOP)**, **FreeRTOS 기반 병렬 처리**, 그리고 **Vector(SIMD) 최적화**를 관통하는 핵심 바이블이 될 것입니다.
## 1. 하드웨어 및 아키텍처 구현 원칙
### [원칙 1] OOP 기반 FSM 오케스트레이션
 * **상태 머신 관리:** FsmManager 클래스를 통해 INIT -> READY -> MONITORING -> RECORDING -> ERROR 상태를 엄격히 제어합니다.
 * **의존성 주입:** 필터, MFCC 엔진, 레코더 객체는 인터페이스를 통해 주입하여 단위 테스트가 가능하도록 설계합니다.
 * **Zero-Hardcoding:** 모든 설정값(샘플링 레이트, 임계값 등)은 Config 클래스나 private static constexpr 상수로 관리하며 매직 넘버 사용을 엄격히 금지합니다.
### [원칙 2] 3단계 RTOS 파이프라인 (Dual-Core 최적화)
 * **Core 0 (Capture):** I2S DMA 수집 및 초동 RMS 계산 전담. (ISR 폭풍 방지를 위해 esp_cpu_get_ccount() 기반 Throttling 적용)
 * **Core 1 (DSP/FSM):** 빔포밍, ANC, MFCC 추출 등 고부하 연산 수행.
 * **Core 1 (Logging):** SD카드 기록 및 네트워크 스트리밍. (DSP 태스크보다 낮은 우선순위)
 * **Task Safety:** 태스크 종료 시 반드시 핸들을 nullptr로 초기화하여 데드락을 방지합니다.
## 2. 메모리 및 데이터 전송 최적화 (Zero-Copy)
### [원칙 3] 메모리 풀 및 인덱스 기반 전송
 * **Zero-Copy 엄수:** 1.2KB 이상의 대형 특징량 구조체는 전역 FeaturePool에 담고, Queue에는 uint8_t slot_idx만 전달하여 SRAM 오버헤드를 최소화합니다.
 * **Bounce Buffer 도입:** PSRAM의 데이터를 SDMMC(DMA)로 직접 밀어 넣지 않고, 내부 SRAM의 바운스 버퍼를 경유하여 캐시 충돌 및 패닉을 방지합니다.
 * **정적 할당:** 텐서 조립용 임시 버퍼는 VLA(가변 길이 배열) 대신 MAX_FEATURE_DIM 상수를 사용해 컴파일 타임에 할당합니다.
### [원칙 4] 데이터 무결성 보존 (Anti-Drop)
 * **Silent Trigger Drop 방지:** MFCC 히스토리(Delta 계산용)가 다 차지 않더라도 Raw 파형 데이터는 즉시 레코더 큐로 전송하여 사고 시점의 초기 데이터를 보존합니다.
 * **Session Ownership:** 파일 닫기 권한은 반드시 소비자(Recorder)에게 부여하여 큐에 남은 마지막 잔여 데이터(Drain)까지 안전하게 기록합니다.
## 3. DSP 및 Vector 연산 최적화 (SIMD)
### [원칙 5] 16-Byte Alignment 및 패딩
 * **SIMD 가속 조건:** esp-dsp 가속기(128-bit)가 4개의 float를 한 번에 읽으므로, 모든 버퍼의 크기는 반드시 **16바이트(4의 배수)**로 패딩하고 MALLOC_CAP_INTERNAL | MALLOC_CAP_8BIT 정렬 할당을 수행합니다.
 * **Ping-pong 버퍼:** In-place(제자리) 연산 오염을 막기 위해 입력과 출력 버퍼를 분리하여 교차 연산합니다.
### [원칙 6] 정밀 필터 파이프라인 순서
 * **순서 엄수:** 위상 왜곡 및 오탐지 방지를 위해 다음 순서를 반드시 준수합니다.
   > **[Median -> DC 제거 -> FIR -> IIR -> Notch -> Beamforming -> ANC]**
   > 
 * **상태 초기화:** 상태가 MONITORING으로 전환될 때 반드시 resetFilterStates()를 호출하여 이전 프레임의 찌꺼기(Delay Line)를 소거합니다.
### [원칙 7] 수치 해석 방어 (IEEE-754)
 * **NaN/Inf 방어:** -ffast-math 최적화 시 isnan()이 무력화될 수 있으므로, 비트 마스킹(0x7F800000)을 통해 하드웨어 레벨에서 직접 검사합니다.
 * **Epsilon 비교:** 부동소수점 0 나누기 방지를 위해 fabsf(v_val) < 1e-6f와 같은 Epsilon 검사를 수행합니다.
## 4. 코딩 컨벤션 및 구현 주의 사항
 * **변수 명명 규칙:**
   * 함수 내부 로컬 변수: v_ 접두사 (예: float v_rmsValue;)
   * 함수 파라미터: p_ 접두사 (예: void process(float* p_inputBuffer);)
 * **Delta 계산:** 시계열 연속성이 깨지는 오버랩 프레임의 경우, 프레임 내 독립 연산을 수행하고 데이터 밀어내기 시에는 반드시 **행 단위(Row-by-row) memcpy**를 사용하여 스트라이드 꼬임을 방지합니다.
 * **네트워크 전송:** 웹소켓 등 전송 시 길이는 반드시 요소 개수 * sizeof(float)를 곱해 정확한 바이트 규격을 맞춥니다.
