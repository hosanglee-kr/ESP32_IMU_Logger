# 📘 SMEA-100 (Super Micro Edge AI) 상세 설계 사양서 (v011_Final)
본 문서는 **v011 리팩토링 마스터플랜**에 따라 전면 개편된 SMEA-100 시스템의 통합 기술 사양서입니다. 실시간 스트리밍 DSP의 수학적 무결성과 MCU의 물리적 한계를 극복하기 위한 모든 설계 원칙을 포함합니다.
## 1. 시스템 핵심 아키텍처 및 철학 (v011 원칙)
### 1.1 엣지 AI 전처리 4대 철학
 1. **연속성 보존(Continuity):** 블록 처리 시 발생하는 프레임 단절(Tearing)을 상태 보존(State Preservation) 변수로 완벽히 차단한다.
 2. **LTI 법칙 준수(Linearity):** 필터의 선형 시불변 특성을 활용하여 채널 병합(Beamforming)을 최상단에 배치, 연산 부하를 50% 절감한다.
 3. **수학적 무결성(Math Integrity):** FFT 에너지 보정(Parseval), DC 바이어스 회피, 복소 Magnitude 연산 등 DSP 교과서적 정석을 엄수한다.
 4. **하드웨어 방어(Hardware Shield):** SIMD 16바이트 정렬, NaN 독성 확산 방지, 부동소수점 흡수 방어 등 MCU 환경의 함정을 코드로 봉쇄한다.
### 1.2 네이밍 및 코드 규격
 * **고정 정수형 강제:** 레지스터 효율과 메모리 안전을 위해 일반 int 사용을 금지하고 <cstdint>(uint8_t, uint16_t, uint32_t)를 전면 적용한다.
 * **변수 스코프 접두어:** 멤버(_), 매개변수(p_), 지역변수(v_)를 엄격히 구분한다.
 * **상수 분류:** 하드웨어 규격은 _CONST, 변경 가능한 기본값은 _DEF 접미사를 사용한다.
## 2. 모듈별 상세 기술 규격
### 2.1 신호 정제 엔진 (T430_DspEng_011)
 * **LTI 최적화 파이프라인:** Beamforming → Median → IIR Notch → FIR HPF → FIR LPF → Pre-emphasis → Noise Gate 순으로 배치한다.
 * **프레임 단절 방어:**
   * _prevPreEmpSample: 이전 프레임의 마지막 샘플(x[-1])을 기억하여 프리엠파시스 팝핑 노이즈를 제거한다.
   * _medianHistory: N/2 크기의 히스토리 버퍼로 Median 필터의 경계면 오염을 방지한다.
 * **스파이크 생성 차단:** 미분기(Pre-emphasis)를 노이즈 게이트 앞단에 배치하여 계단파 미분에 의한 델타 스파이크 발생을 원천 차단한다.
 * **DC 제거 전략:** 파괴적인 블록 평균 차단 로직을 삭제하고, 위상 선형성이 보장된 FIR HPF에 전적으로 의존한다.
### 2.2 특징량 추출 엔진 (T440_FeatExtra_011)
 * **수학적 보정 및 교정:**
   * **Parseval 에너지 복원:** FFT 결과에 단방향 보정(\times 2.0) 및 윈도우 감쇠 보정(\times 2.666) 계수를 적용하여 절대 에너지를 보존한다.
   * **EMA 노이즈 학습:** 누적 평균의 부동소수점 흡수 현상을 막기 위해 지수이동평균(EMA) 알고리즘으로 ANC 프로파일을 생성한다.
   * **NMS 피크 추출:** min_freq_gap_hz 내의 하위 피크를 강제 탈락시키는 Non-Maximum Suppression으로 스펙트럼 누출(Side-lobe)을 방어한다.
   * **슬라이딩 STA/LTA:** 1024 샘플 전체를 $O(N)$으로 스캔하여 프레임 내 어느 지점에서 발생한 충격음도 놓치지 않고 캡처한다.
 * **켑스트럼(Cepstrum) 정밀화:**
   * 주파수(Hz)를 주기(Sec)로 역산 후 샘플링 레이트를 곱해 정확한 Quefrency 인덱스를 도출한다.
   * max(1, startIdx) 방어로 0Hz(DC) 성분이 피크로 오인되는 버그를 차단한다.
   * 실수부뿐만 아니라 \sqrt{Re^2 + Im^2} 로 복소수 크기 에너지를 산출한다.
 * **공간 위상 분석:** 위상 보존을 위해 L/R 채널 모두에 동일한 Hanning Window를 적용한 후 IPD(위상차) 및 Coherence를 분석한다.
### 2.3 시스템 관리 및 통신 (T450_Fsm, T420_Types, T470_Commu)
 * **데이터 평탄화(Flattening):** PktTelemetry 구조체 내의 top_peaks를 주파수/진폭 배열로 분리하여 JS Float32Array 파싱 정합성을 확보한다.
 * **멀티플렉싱 바이너리:** WebSocket 헤더(0xA5, Type, Length)를 통해 텔레메트리(0x01), 스펙트럼(0x02), 파형(0x03)을 지능적으로 분산 전송한다.
 * **이중 큐 풀링:** 100개 규모의 슬롯 풀을 운영하며, xQueueSend 실패 시 즉시 인덱스를 반환하는 롤백(Rollback) 로직으로 메모리 고갈을 방지한다.
 * **원자적 설정(Atomic Config):** /api/runtime_config 수신 시 임시 파일(tmp) 쓰기 후 이름 변경(Rename) 방식으로 정전 시 파일 파손을 방어한다.
## 3. 웹 인터페이스 (Web UI) 구현 규격
### 3.1 초고속 렌더링 아키텍처
 * **uPlot 기반 렌더링:** DOM 부하를 최소화하기 위해 Canvas 기반의 uPlot을 사용하여 100Hz 급의 시계열 데이터를 60fps로 시각화한다.
 * **디커플링 루프:** WebSocket 수신부(onmessage)는 데이터 갱신만 수행하고, 실제 화면 그리기는 브라우저의 requestAnimationFrame 주기에 맞춘다.
 * **바이너리 캐스팅:** DataView와 Float32Array를 사용하여 JSON 파싱 없이 수신 즉시 메모리 포인터를 차트 데이터로 매핑한다.
### 3.2 사용자 편의 기능
 * **시뮬레이션 모드:** 하드웨어 없이도 UI 로직을 검증할 수 있는 100Hz 가상 데이터 생성 엔진(Mock Engine)을 탑재한다.
 * **하이브리드 대시보드:**
   * **Waterfall:** MFCC 39차원 변화량을 히트맵 형태로 누적 표시한다.
   * **Peak Overlay:** 스펙트럼 차트 위에 실시간으로 추출된 Top-5 피크를 동적으로 표기한다.
## 4. 안정성 및 보안 방어벽 (21대 원칙 요약)
 1. **SIMD NaN 독성 차단:** FFT 패딩 공간을 memset 0으로 초기화하여 쓰레기값에 의한 전역 NaN 오염을 막는다.
 2. **부동소수점 쉴드:** fmaxf(val, EPSILON)를 모든 로그, 루트, 나눗셈 분모에 적용한다.
 3. **컴파일 타임 검증:** static_assert를 사용하여 FIR 탭 수가 짝수로 설정되어 필터가 붕괴하는 것을 빌드 단계에서 막는다.
 4. **포인터 Immutability:** 모든 입력 파형 포인터는 const로 잠가 원본 데이터가 DSP 필터에 의해 오염되는 것을 방지한다.
