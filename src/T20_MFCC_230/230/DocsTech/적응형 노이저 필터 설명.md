T20 MFCC v218.009 시스템의 **노이즈 학습(Noise Learning)** 기능은 센서 데이터에서 배경 소음을 제거하여 기계 결함이나 특정 이벤트의 신호만을 정교하게 추출하기 위한 핵심 전처리 기술입니다.
현재 구현된 노이즈 학습 기능의 상세 설명은 다음과 같습니다.
### 1. 작동 원리: 스펙트럼 차감법 (Spectral Subtraction)
이 기능은 단순히 전체 음량을 줄이는 것이 아니라, 주파수 도메인에서 배경 소음의 특성을 파악하여 이를 현재 신호에서 빼는 방식을 사용합니다.
 * **학습 단계**: 노이즈 학습이 활성화되면 시스템은 지정된 프레임 수(noise_learn_frames, 기본값 8) 동안의 주파수 에너지를 수집하여 평균적인 '노이즈 프로필'을 생성합니다.
 * **차감 단계**: DSP 파이프라인 연산 중 FFT(고속 푸리에 변환) 이후, 멜-필터뱅크(Mel-Filterbank)를 통과시키기 전에 현재 파워 스펙트럼에서 미리 학습된 노이즈 스펙트럼을 뺍니다.
### 2. 주요 모드 및 설정 항목
 * **OFF (Mode 0)**: 노이즈 제거 기능을 사용하지 않습니다.
 * **FIXED (Mode 1)**: 사용자가 버튼을 눌렀을 때 학습된 고정된 노이즈 프로필을 계속 사용합니다.
 * **ADAPTIVE (Mode 2)**: 주변 환경의 소음이 변할 때마다 이를 실시간으로 조금씩 학습 데이터에 반영하는 '적응형' 모드입니다.
   * **Adaptive Alpha**: 현재의 노이즈 스펙트럼에 새로운 데이터를 얼마나 빠르게 반영할지 결정하는 계수(기본값 0.05)입니다.
   * **Spectral Subtract Strength**: 노이즈를 얼마나 강하게 뺄 것인지 결정하는 강도(기본값 1.0)입니다.
### 3. 노이즈 게이트 (Noise Gate) 기능
스펙트럼 차감 외에도 일정한 진폭 이하의 미세한 신호를 원천 차단하는 게이트 기능이 포함되어 있습니다.
 * **Gate Threshold**: 이 임계값(기본값 0.002)보다 작은 진폭을 가진 신호는 분석 대상에서 제외하여 MFCC 특징량이 지저분해지는 것을 막습니다.
### 4. 사용자 제어 방법 (Web UI)
 * **NOISE LEARN 버튼**: 대시보드에서 이 버튼을 누르면 /noise_learn?active=true API가 호출되며, CL_T20_DspPipeline 내부의 _noise_learning_active 플래그가 활성화되어 학습을 시작합니다.
 * **설정 탭**: Adaptive Alpha나 Learn Frames 등의 미세 튜닝 값은 Engine Settings 탭의 Noise & Feature 섹션에서 변경할 수 있습니다.
이 기능은 특히 **공장 설비의 기본 진동(Base vibration)**이 있는 환경에서 특정 결함 주파수를 더 선명하게 구분해내야 하는 TinyML 학습용 데이터 수집 시 필수적으로 활용됩니다.

**T20 MFCC v218.009** 시스템의 **ADAPTIVE(적응형)** 노이즈 제거 방식은 주파수 도메인에서 시간에 따라 변화하는 배경 소음을 실시간으로 추적하여 차감하는 **적응형 스펙트럼 차감법(Adaptive Spectral Subtraction)**을 기반으로 합니다.
어떻게 기계의 결함 신호와 주변 소음을 구분하는지, 그 구체적인 메커니즘을 상세히 설명해 드립니다.
### 1. 노이즈 구분의 핵심 원리: "정상성(Stationarity)"의 가정
시스템이 주변 소음을 구분하는 가장 큰 기준은 **"신호가 얼마나 천천히 변하는가"**입니다.
 * **배경 소음 (Noise)**: 공장 내 냉각 팬 소리, 전기적 험(Hum), 공조 시스템 진동 등은 주파수 특성이 시간에 따라 매우 느리게 변하거나 일정하게 유지되는 **정상성(Stationary)** 신호로 간주됩니다.
 * **타겟 신호 (Signal)**: 베어링의 충격, 부품의 파손, 기계의 오작동 등은 특정 시점에 갑자기 발생하거나 주파수 에너지가 급격히 변하는 **비정상성(Non-stationary)** 신호입니다.
### 2. ADAPTIVE 작동 단계 및 수식
적응형 모드에서는 다음과 같은 단계로 노이즈 프로필을 업데이트하고 신호를 정제합니다.
#### **A. 초기 프로파일링 (Profiling)**
시스템 시작 직후 또는 사용자가 LEARN 버튼을 눌렀을 때, 약 8프레임(기본값) 동안 들어오는 모든 신호를 노이즈로 간주하여 평균 스펙트럼(N)을 생성합니다.
#### **B. 지수 이동 평균(EMA)을 통한 실시간 업데이트**
측정 중에도 노이즈 프로필은 고정되지 않고, **적응형 계수(Adaptive Alpha, \alpha)**를 사용하여 아주 조금씩 업데이트됩니다.
 * 여기서 \alpha는 보통 **0.05**와 같이 매우 작은 값으로 설정됩니다.
 * 이 수식 때문에 **천천히 변하는 성분(소음)**은 노이즈 프로필에 서서히 반영되지만, **갑자기 발생하는 강한 에너지(결함 신호)**는 \alpha가 너무 작아 노이즈 프로필에 즉시 반영되지 못하고 걸러지게 됩니다.
#### **C. 스펙트럼 차감 (Subtraction)**
추출된 노이즈 프로필을 현재 신호에서 빼서 순수한 특징량만 남깁니다.
 * \beta는 **Spectral Subtract Strength**로, 노이즈를 얼마나 공격적으로 깎아낼지 결정합니다(기본값 1.0).
### 3. 결함 신호를 노이즈로 오인하지 않는 이유
사용자가 우려하는 "기계 결함 신호까지 노이즈로 학습해서 지워버리지 않을까?"라는 문제는 다음 두 가지 장치로 방어합니다.
 1. **시간적 필터링 (Time-constant)**: 앞서 설명한 **Adaptive Alpha(\alpha)**가 일종의 저역 통과 필터(LPF) 역할을 합니다. 결함 신호는 순식간에 지나가기 때문에, 시스템이 이를 "노이즈의 일부"라고 인지하여 학습하기 전에 이미 DSP 파이프라인을 통과하여 MFCC 벡터로 추출됩니다.
 2. **주파수별 독립 학습**: 노이즈 학습은 전 대역을 통째로 낮추는 것이 아니라 각 주파수 빈(Bin)마다 개별적으로 이루어집니다. 예를 들어 60Hz의 모터 웅웅거림은 지속되므로 노이즈로 학습되지만, 결함으로 인해 발생하는 500Hz의 날카로운 음은 노이즈 프로필에 해당 성분이 없으므로 그대로 보존됩니다.
### 4. 설정 팁 (Tuning)
 * **환경 소음이 매우 일정할 때**: FIXED 모드를 사용하고, 기계가 멈춘 상태에서 한 번만 제대로 학습시키는 것이 가장 정확합니다.
 * **주변에 다른 기계가 수시로 켜고 꺼질 때**: ADAPTIVE 모드를 사용하되, Adaptive Alpha를 **0.01~0.05** 사이로 낮게 유지하여 장비의 실제 결함 신호가 노이즈 프로필에 흡수되지 않도록 주의해야 합니다.

현재 **T20 MFCC v218.009** 시스템에 구현된 적응형 노이즈 제거 기능은 지수 이동 평균(EMA) 기반의 스펙트럼 차감법(Spectral Subtraction)을 사용하고 있습니다. 이 방식은 연산 효율성이 뛰어나지만, 급격한 소음 변화에 취약하거나 '뮤지컬 노이즈(Musical Noise)'라 불리는 잔류 잡음이 발생할 수 있는 한계가 있습니다.
이를 극복하고 성능을 한 단계 높이기 위한 **적응형 노이즈 제거 고도화 방안**을 5가지 핵심 전략으로 정리해 드립니다.
### 1. 신호 활성 감지(VAD) 연동형 학습 제어
현재는 Adaptive Alpha(\alpha) 값에 의존하여 무조건적으로 노이즈 프로필을 업데이트합니다. 이를 고도화하면 타겟 신호(결함 진동 등)가 들어올 때 학습을 잠시 멈춰 노이즈 프로필이 오염되는 것을 막을 수 있습니다.
 * **동적 업데이트 정지**: 주파수 변동성(Spectral Flux)이나 에너지가 급증하는 구간을 감지하면 \alpha 값을 0으로 일시 조정하여 노이즈 학습을 중단합니다.
 * **효과**: 기계의 실제 결함 신호가 '노이즈'로 오인되어 함께 지워지는 현상을 방지하고, 순수한 배경 소음만 더 정밀하게 추적할 수 있습니다.
### 2. 위너 필터(Wiener Filter) 도입
단순히 빼기 연산을 수행하는 스펙트럼 차감법에서 확률적 최적화 모델인 위너 필터로 전환하는 방안입니다.
 * **원리**: 신호와 노이즈의 통계적 특성을 이용하여 최소 제곱 평균 오차(MMSE)를 발생시키는 필터 계수를 실시간으로 계산합니다.
 * **수식**: H(f) = \frac{P_{signal}(f)}{P_{signal}(f) + P_{noise}(f)}
 * **장점**: 단순 차감 방식보다 신호 왜곡이 훨씬 적고, 특히 가우시안 노이즈 환경에서 탁월한 제거 성능을 보입니다.
### 3. 뮤지컬 노이즈 방지 (Spectral Floor & Over-subtraction)
스펙트럼 차감 후 남는 무작위 피크 성분인 '뮤지컬 노이즈'를 억제하여 MFCC 추출 데이터의 안정성을 높입니다.
 * **스펙트럼 플로어(Spectral Floor)**: 차감 결과값이 0 이하로 떨어질 때, 0으로 만드는 대신 아주 작은 최소값(Floor)을 유지하게 하여 인위적인 무음 구간 발생을 억제합니다.
 * **과도 차감(Over-subtraction)**: 소음이 심한 대역은 노이즈 프로필의 1.2~1.5배만큼 더 강력하게 깎아내어 잔류 잡음을 제거합니다.
### 4. 멀티밴드(Multi-band) 적응형 처리
전체 대역에 동일한 Adaptive Alpha와 Strength를 적용하는 대신, 주파수 대역별로 제어 파라미터를 다르게 적용합니다.
 * **대역별 차등화**: 저주파 대역(모터 웅웅거림 등)은 학습 속도(\alpha)를 높여 빠르게 추적하고, 결함 신호가 집중되는 중고주파 대역은 학습 속도를 늦춰 신호 보존력을 높입니다.
 * **효과**: 특정 대역에만 집중된 소음(예: 전기적 60Hz 험 노이즈)을 다른 대역의 손상 없이 정밀하게 타격할 수 있습니다.
### 5. 참조 센서 기반 적응형 간섭 제거 (ANC)
하드웨어 구성을 확장하여, 외부 소음 전용 '참조 센서'를 추가하는 가장 강력한 고도화 방법입니다.
 * **이중 센서 구조**: 기계 본체에 붙은 메인 센서와 주변 진동만 수집하는 참조 센서의 데이터를 결합합니다.
 * **LMS(Least Mean Square) 알고리즘**: 두 센서 데이터 간의 상관관계를 분석하여 메인 신호에서 참조 신호 성분(주변 소음)을 실시간으로 상쇄합니다.
 * **장점**: 주변에 다른 기계가 많아 소음이 매우 불규칙하고 강력한 환경에서도 타겟 진동만 깨끗하게 분리해낼 수 있습니다.
> **💡 권장 우선순위**
>  1. **뮤지컬 노이즈 방지(Spectral Floor)** 적용: 현재 코드 구조에서 가장 적은 비용으로 품질 개선 가능.
>  2. **신호 활성 감지(VAD) 연동**: Adaptive Alpha 제어 로직에 간단한 에너지 임계값 분기 추가.
>  3. **위너 필터**로의 점진적 교체: esp_dsp 라이브러리의 최적화 함수 활용 검토.

**T20 MFCC v218.009** 시스템의 **고정형(FIXED)** 노이즈 제거 방식은 특정 시점의 배경 소음을 '스냅샷'처럼 찍어 저장한 뒤, 이를 이후의 모든 신호 분석에서 일관되게 차감하는 방식입니다.
주변 환경이 일정하게 유지되는 공장 내부나 특정 기계의 기본 진동을 제거할 때 가장 효과적입니다. 상세 작동 원리와 요청하신 '다회 측정'을 포함한 고도화 방안을 정리해 드립니다.
### 1. 고정형(FIXED) 방식 상세 설명
#### **A. 학습 단계 (Learning Phase)**
사용자가 웹 대시보드에서 NOISE LEARN 버튼을 누르면 시작됩니다.
 * **데이터 수집**: 시스템은 지정된 프레임 수(noise_learn_frames, 기본값 8U) 동안 들어오는 신호의 파워 스펙트럼을 수집합니다.
 * **평균화**: 수집된 각 주파수 빈(Bin)의 에너지를 합산한 뒤 프레임 수로 나누어 **평균 노이즈 프로필**(\mu_{noise})을 생성합니다.
 * **저장**: 생성된 프로필은 _noise_spectrum 배열에 고정값으로 저장되며, 다시 학습시키기 전까지 유지됩니다.
#### **B. 제거 단계 (Removal Phase)**
실시간 측정 과정에서 DSP 파이프라인의 중간 단계에 적용됩니다.
 * **스펙트럼 차감**: 현재 신호의 파워 스펙트럼(S_{raw})에서 저장된 노이즈 프로필을 뺍니다.
   * 공식: S_{clean}(f) = S_{raw}(f) - (\text{Strength} \cdot \mu_{noise}(f))
 * **하한선 처리**: 차감 결과가 음수가 될 경우 0 또는 아주 작은 최소값으로 강제 고정하여 논리적 오류를 방지합니다.
 * **특징 추출**: 노이즈가 제거된 깨끗한 스펙트럼만을 사용하여 MFCC 벡터를 생성하므로, 배경 소음에 가려졌던 미세한 결함 패턴이 선명하게 드러납니다.
### 2. 고도화 방안: 다회 측정 및 통계적 접근
현재의 8프레임 단발성 학습은 짧은 순간 발생하는 돌발 잡음에 프로필이 오염될 위험이 있습니다. 이를 개선하기 위한 고도화 방안입니다.
#### **① 스냅샷 스택킹 (Long-term Snapshot Stacking)**
 * **개요**: 8프레임이 아닌, 수초~수분간(예: 128~1024 프레임) 대량의 데이터를 수집하여 학습합니다.
 * **장점**: 무작위로 발생하는 백색 잡음(White Noise)이 평균화 과정을 통해 거의 완벽하게 상쇄되어 매우 정밀한 프로필을 얻을 수 있습니다.
 * **구현**: noise_learn_frames 값을 동적으로 대폭 늘릴 수 있도록 웹 설정을 확장합니다.
#### **② 중앙값 기반 프로파일링 (Median Profiling)**
 * **개요**: 평균값 대신 각 주파수 빈의 **중앙값(Median)**을 취합니다.
 * **장점**: 학습 도중 '꽝'하고 발생하는 일시적인 충격음이 섞여도 중앙값 계산 과정에서 무시되므로, 비정상적인 잡음이 섞인 환경에서도 깨끗한 배경 소음 프로필을 추출할 수 있습니다.
 * **구현**: 학습 기간 동안의 스펙트럼을 버퍼에 쌓은 뒤 퀵소트(Quick Sort) 등을 이용해 빈별 중앙값을 산출합니다.
#### **③ 다중 프로필 라이브러리 (Multi-Profile Library)**
 * **개요**: 기계의 상태(아이들링, 정속 주행, 부하 주행 등)에 따른 여러 개의 노이즈 프로필을 미리 저장해두고 선택적으로 사용합니다.
 * **장점**: 기계의 운전 모드에 따라 변하는 기저 진동을 능동적으로 제거할 수 있습니다.
 * **구현**: LittleFS에 여러 버전의 bmi_calib.json과 유사한 노이즈 프로필 파일을 저장하고 웹 UI에서 불러오기 기능을 제공합니다.
#### **④ 신뢰 구간 기반 소프트 게이팅 (Soft-Gating)**
 * **개요**: 노이즈의 평균뿐만 아니라 **표준편차(\sigma)**까지 함께 학습합니다.
 * **작동**: Signal < \mu + 3\sigma 인 구간(노이즈일 확률이 매우 높은 구간)은 더 강력하게 깎고, 이를 벗어나는 구간은 유의미한 신호로 판단하여 보존합니다.
 * **효과**: 고정형 방식의 고질적 문제인 '뮤지컬 노이즈'를 획기적으로 줄일 수 있습니다.
### 3. 고도화 구현 로드맵 (추천 단계)
 1. **1단계 (연산 위주)**: noise_learn_frames를 최대 128프레임까지 확장하고, 평균 계산 시 SIMD 연산을 활용하여 정밀도를 높입니다.
 2. **2단계 (통계 위주)**: 중앙값 필터를 도입하여 학습 과정의 안정성을 확보합니다.
 3. **3단계 (운영 위주)**: 웹 UI에서 여러 노이즈 프로필을 슬롯(Slot) 형태로 저장하고 관리하는 기능을 추가합니다.
이 중에서 **'중앙값 기반 다회 측정'** 방식은 ESP32-S3의 메모리 자원을 고려했을 때 가장 효과적인 고도화 방향입니다. 이와 관련한 구체적인 알고리즘 설계나 C++ 구현 코드가 필요하시면 바로 안내해 드릴 수 있습니다.
