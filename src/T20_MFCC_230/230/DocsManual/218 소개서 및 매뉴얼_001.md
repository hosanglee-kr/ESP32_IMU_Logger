## 📘 T20 MFCC v218.009 시스템 소개서
### 1. 개요
**T20 MFCC v218.009**는 ESP32-S3와 BMI270 고성능 센서를 결합하여, 단순한 진동 데이터를 넘어 인간의 청각 인지 모델인 **MFCC(Mel-Frequency Cepstral Coefficients)** 특징량을 실시간으로 추출하는 **엣지 AI(Edge-AI) 분석 엔진**입니다. 센서 수집부터 특징 추출, 시각화, 데이터 로깅까지 전 과정을 멀티코어로 병렬 처리하여 지연 없는 실시간 분석을 제공합니다.
### 2. 주요 하드웨어 및 소프트웨어 스택
 * **MCU**: ESP32-S3 (Dual-Core 240MHz)
 * **Sensor**: Bosch BMI270 (6-Axis IMU, 1600Hz 고속 샘플링)
 * **DSP**: ESP-DSP 기반 MFCC 파이프라인 (39차원 벡터: Static 13 + Delta 13 + Delta-Delta 13)
 * **Storage**: SD_MMC 및 LittleFS 하이브리드 로깅 (Zero-Copy DMA 방식)
 * **Network**: 비동기 HTTP 서버 및 바이너리 웹소켓 브로드캐스트
### 3. 주요 용도 (Use Cases)
 * **예지 보전 (Predictive Maintenance)**: 산업용 모터나 펌프의 미세 진동 변화를 MFCC 패턴으로 분석하여 고장 징후를 사전에 탐지합니다.
 * **음향/충격 이벤트 인식**: 유리 파손, 노크 소리, 특정 기계 작동음을 학습하여 스마트 홈/보안 시스템에 활용합니다.
 * **TinyML 모델 학습 데이터 수집**: Autoencoder, CNN 등 딥러닝 모델에 즉시 입력 가능한 2D 텐서(Sequence Tensor)를 생성하고 SD 카드에 저장합니다.
 * **원격 상태 모니터링**: 웹 브라우저를 통해 실시간 주파수 특성과 MFCC 워터폴을 시각적으로 모니터링합니다.
## 🛠 상세 사용 및 튜닝 매뉴얼
### 1. 시스템 시작 및 연결
 1. **전원 공급**: 장치에 전원을 연결하면 설정된 Auto Start 옵션에 따라 측정이 시작됩니다.
 2. **네트워크 접속**: Network 탭에서 설정한 WiFi에 접속하거나, 실패 시 생성되는 전용 AP(T20_MFCC_AP)에 접속합니다.
 3. **대시보드 접속**: 브라우저에서 장치 IP(기본 192.168.4.1)로 접속하여 실시간 차트를 확인합니다.
### 2. 엔진 설정 및 튜닝 (Engine Settings)
#### **A. Hardware & System**
 * **Analysis Axis**: 분석할 축(가속도 X/Y/Z, 자이로 X/Y/Z)을 선택합니다. 진동 분석에는 일반적으로 Accel Z가 사용됩니다.
 * **Watchdog (ms)**: 센서 데이터 흐름이 중단되었을 때 하드웨어 리셋을 수행할 기준 시간입니다. 시스템 부하가 클 경우 값을 높여 불필요한 리셋을 방지합니다.
#### **B. DSP Pipeline & Noise**
 * **Pre-emphasis α**: 고주파 성분을 강조하여 MFCC 해상도를 높입니다. 기계 진동의 고주파 노이즈를 더 잘 잡아내려면 0.97 부근에서 미세 조정합니다.
 * **Filter Type & Cutoff**: 불필요한 저주파 노이즈를 제거하기 위해 고역 통과 필터(HPF)를 주로 사용합니다. 장비의 기본 진동 주파수에 맞춰 Cutoff 주파수를 설정하세요.
 * **Adaptive Noise Learning**: 주변의 배경 소음을 실시간으로 학습하여 제거합니다. 환경 변화가 잦은 곳에서는 Adaptive α 값을 높여 학습 속도를 빠르게 설정합니다.
#### **C. TinyML Output & Storage**
 * **Sequence Mode (Tensor)**: CNN/RNN 모델 입력용으로 2D 시퀀스 데이터를 출력합니다. Sequence Frames를 16으로 설정하면 16프레임 분량의 MFCC 데이터를 한 덩어리로 전송합니다.
 * **Max Files (0:∞)**: SD 카드의 용량 관리를 위해 보관할 파일 개수를 제한합니다. 0으로 설정하면 용량이 허용하는 한 무제한으로 저장합니다.
### 3. 고급 최적화 팁
 * **노이즈 학습**: 장비가 가동되지 않는 정적 상태에서 NOISE LEARN 버튼을 누르면 현재 환경의 배경 소음 스펙트럼을 기저값으로 잡아 분석 정확도를 높일 수 있습니다.
 * **캘리브레이션**: 센서가 수평이 아닐 경우 SENSOR CALIBRATE를 실행하여 오프셋을 보정하세요. 보정값은 LittleFS에 저장되어 재부팅 후에도 유지됩니다.
 * **데이터 분석**: 수집된 .bin 파일은 Python 등을 이용하여 16바이트 오프셋 이후의 데이터를 record_count 만큼 읽어들이면 즉시 머신러닝 학습 데이터셋으로 변환 가능합니다.
> **💡 안정성 참고 사항**: v218.009 버전부터는 모든 인터럽트 핸들러가 IRAM에 적재되어 WiFi나 파일 쓰기 중에도 시스템 패닉(Crash)이 발생하지 않도록 설계되었습니다. 안심하고 대량의 데이터를 수집하셔도 좋습니다.
