### 2. 설비 상태 및 속성 제어 구조체 설계 (C++)

  상태 번호 체계와 세분화된 가감속/반복 패턴을 반영한 구조체입니다.
```cpp
#pragma once

#include <stdint.h>

/**
 * @brief Level 1: 상태 대분류 (10 단위 증가)
 */
enum EM_T20_L1_State {
    ST_L1_DOWN		= 10,
    ST_L1_IDLE		= 20,
    ST_L1_RUN		= 30,
    ST_L1_UNKNOWN	= 90
};

/**
 * @brief Level 2: 운영 유형 (10 단위 증가)
 */
enum EM_T20_L2_Type {
	ST_L2_NONE               = 100,
	ST_L2_IDLE               = 210,
	ST_L2_IDLE_UNKNOWN       = 290,
	ST_L2_STEADY_RUN         = 310,
	ST_L2_DYN_ACC            = 320,
	ST_L2_DYN_DEC            = 330,
	ST_L2_DYN_CYCLIC         = 340,
	ST_L2_DYN_COASTINGDOWN   = 380,
	ST_L2_RUN_UNKNOWN        = 390,
	ST_L2_UNKNOWN            = 900
};

/**
 * @brief Level 3: 세부 패턴 (10 단위 증가)
 */
enum EM_T20_L3_Detail {
	ST_L3_DOWN              = 1000,
	
    // IDLE Details
    ST_L3_IDLE_QUIET        = 2110,
    ST_L3_IDLE_ACTIVE       = 2120,
    ST_L3_IDLE_UNKNOWN      = 2900,

    // RUN Steady Details
    ST_L3_RUN_SPD_LOW       = 3110,
    ST_L3_RUN_SPD_MID       = 3120,
    ST_L3_RUN_SPD_HIGH      = 3130,

    // RUN Dynamic Details (Acc/Dec/Cyclic)
    ST_L3_DYN_ACC_LOW       = 3210, 
    ST_L3_DYN_ACC_MID       = 3220, 
    ST_L3_DYN_ACC_HIGH      = 3230,
	
    ST_L3_DYN_DEC_LOW       = 3310, 
    ST_L3_DYN_DEC_MID       = 3320, 
    ST_L3_DYN_DEC_HIGH      = 3330,
    
    ST_L3_DYN_CYC_LOW       = 3410, 
    ST_L3_DYN_CYC_MID       = 3420, 
    ST_L3_DYN_CYC_HIGH      = 3430,
    
    ST_L3_DYN_COASTING      = 3800,
	
    ST_L3_RUN_UNKNOWN       = 3900,
    
    // System Unknown
    ST_L3_UNKNOWN           = 9000
};

/**
 * @brief 독립적 속성 (중복 허용 비트 플래그)
 */
struct ST_T20_Attributes {
    uint16_t          is_loaded     : 1;  // 부하 상태
    uint16_t          is_resonance  : 1;  // 공진 발생
    uint16_t          is_anomaly    : 1;  // 이상 징후 (핵심 이벤트)
    uint16_t          is_noisy      : 1;  // 환경 노이즈
    uint16_t          is_clipping   : 1;  // 센서 포화(Clipping)
};

/**
 * @brief T20 지능형 상태 관리 컨텍스트
 */
struct ST_T20_MachineContext {
    EM_T20_L1_State   l1;
    EM_T20_L2_Type    l2;
    EM_T20_L3_Detail  l3;
    ST_T20_Attributes attr;

    // 25프레임(4초) 분석용 내부 변수
    float             dF_dt_slope;       // 가감속 등급 판별 (Slope)
    float             cyclic_freq_hz;    // 반복 빈도 판별 (Frequency)
    float             last_rms;
    
    // 디바운스 카운터
    uint16_t          confirm_cnt;
    uint16_t          transition_cnt;
};

```
### 💡 주요 설계 포인트 및 구현 전략
 1. **가감속/반복 3단계 구분:**
 * **Acc/Dec (dF/dt):** 25프레임 윈도우 내의 주파수 변화 기울기를 계산하여, Slope < th1(Low), th1 <= Slope < th2(Mid), Slope >= th2(High)로 분류합니다.
 * **Cyclic (빈도):** 주파수가 요동치는 주기(Period)를 측정합니다. 1초에 1회 미만 반복 시 **Low**, 1~3회 **Mid**, 3회 이상 **High** 빈도로 정의합니다.
 
 2. **Coasting Down 판별:**
 * 전원이 차단된 직후 주파수가 지수 함수적으로 급격히 감소하는 특성을 log(Freq)의 선형성 검사를 통해 감속(Deceleration) 상태와 구분해 냅니다.
 
 3. **라벨링 최소화 (Normal Baseline):**
 * **3120 (Speed Mid)** 상태이면서 attr.is_anomaly == 0인 상태는 자동으로 **'정상 중속 운전'** 데이터로 분류됩니다.
 * 모든 **3900 (Run Unknown)** 계층과 is_anomaly == 1인 데이터만 관리자가 확인하게 함으로써 현장의 라벨링 리소스를 90% 이상 절감합니다.

  이 구조는 데이터 저장 시 파일명에 30_320_3230_04.bin (RUN_ACC_HIGH_IsNoisy)과 같이 상태 코드를 직접 매핑할 수 있어 관리가 매우 편리합니다.

## II. 설비 상태 별 판단 기준
*** *** *** *** *** *** *** *** *** *** *** *** *** *** *** *** *** *** *** *** 

  확정된 **상태 번호 체계(Level 1~3)**를 바탕으로, 각 상태를 칼같이 분리하기 위한 **판단 기준 정교화(Judgment Logic Refinement)** 작업을 진행합니다.  
  MECE 원칙을 준수하기 위해 모든 판단은 **상위 계층에서 하위 계층으로 내려가는 '필터링 방식'**을 채택하며, 4초(25프레임) 데이터의 물리적 특성을 수학적으로 정의합니다.

### 1. 전역 판단 변수(Constants) 및 추가 피처 정의
  정교한 분리를 위해 기존 피처 외에 **3가지 보완 피처**를 추가 검토 및 적용합니다.

 * **R-Square (결정계수):** 25프레임 주파수 추세의 **선형성** 평가. (동적 가/감속 vs 불규칙 변동 구분용)
 * **Zero-Crossing Rate of dF/dt:** 주파수 기울기의 **부호 변화 빈도**. (Cyclic 판별 핵심)
 * **Log-Linearity Fit:** 주파수 감소 패턴이 **지수적(Exponential)**인지 확인. (Coasting Down 구분용)

### 2. 단계별 MECE 판단 로직 상세 설계

#### 2.1 Step 1) L1: 존재 에너지 기반 분리 (RMS Filter)
  가장 먼저 설비가 살아있는지(L1)를 결정합니다.

| L1 상태 | 판단 기준 |
|---|---|
| **10 (DOWN)** | RMS < RMS_Floor **AND** Spectral_Flatness > 0.8 |
| **20 (IDLE)** | RMS_Floor <= RMS < RMS_Run_Min |
| **30 (RUN)** | RMS >= RMS_Run_Min |
| **90 (UNKNOWN)** | 위 조건이 충돌하거나 센서 통신 에러 시. |

#### 2.2 Step 2) L2: 주파수 안정성 기반 분리 (Stability Filter)
  L1이 **30 (RUN)**일 때, 주파수 추세(Slope)를 분석합니다.

| L1 | L2 | 판단 기준 |
|---|---|---|
| **30 (RUN)** | **310 (Steady RUN)** | abs(dF_dt_slope) < Stability_Threshold (기울기가 거의 0) |
| **30 (RUN)** | **320 (Dynamic Acc)** | dF_dt_slope >= Stability_Threshold |
| **30 (RUN)** | **330 (Dynamic Dec)** | dF_dt_slope <= -Stability_Threshold |
| **30 (RUN)** | **340 (Dynamic Cyclic)** | Cyclic_Indicator > High (지속적 부호 반전) |
| **30 (RUN)** | **380 (Coasting Down)** | Log-Linearity 기준 충족 (지수적 감쇠) |
| **30 (RUN)** | **390 (Run Unknown)** | 에너지(RMS)는 높으나 주파수 패턴을 상기 범주로 특정할 수 없는 경우. |

#### 2.3 Step 3) L3: 세부 패턴 및 등급 분리 (Pattern & Grade Filter)

가장 복잡한 Dynamic RUN(동적 운전) 상태와 정속 상태를 등급별로 쪼갭니다.

| 분류 대상 (L2) | 상세 상태 (L3 ID) | 정교화된 판단 공식 (MECE Logic) |
|---|---|---|
| **310 (Steady)** | **3110 / 3120 / 3130** | Target_Freq가 사용자 정의 구간(f1, f2) 중 어디에 속하는가로 결정 |
| **320 (Acc)** | **3210 / 3220 / 3230** | Slope > 0 인 상태에서 기울기 값에 따라 분리 (예: Low < 5Hz/s < Mid < 15Hz/s < High) |
| **330 (Dec)** | **3310 / 3320 / 3330** | Slope < 0 인 상태에서 기울기 값에 따라 분리 (Power-on 상태 유지 전제) |
| **340 (Cyclic)** | **3410 / 3420 / 3430** | Slope의 부호 반전 횟수가 4초 내에 N회 이상 발생 시 빈도에 따라 분리 |
| **380 (Coasting)** | **3800 (Coasting Down)** | Slope < 0 **AND** RMS가 동시에 지수적으로 급격히 감소하는 패턴 일치 시 |

### 3. 상태별 판단 기준 매트릭스 (실무 파라미터)

| 상태 ID | 판단 피처 1 (주지표) | 판단 피처 2 (보조지표) | MECE 경계값 (예시) |
|---|---|---|---|
| **1000 (Down)** | **RMS** | Spectral Flatness | RMS < 0.01G AND Flatness > 0.85 |
| **2110 (Idle Quiet)** | **RMS** | Peak Amplitude | 0.01G < RMS < 0.05G AND Peak < 0.03G |
| **3110~3130 (Speed L/M/H)** | **dF/dt Slope** | Peak Frequency | abs(Slope) < 0.5Hz/s (25프레임 평균) |
| **3210~3230 (Acc L/M/H)** | **dF/dt Slope** | R-Square (선형성) | Slope > 0.5Hz/s AND R2 > 0.7 |
| **3310~3330 (Dec L/M/H)** | **dF/dt Slope** | R-Square (선형성) | Slope < -0.5Hz/s AND R2 > 0.7 |
| **3410~3430 (Cyclic L/M/H)** | **Zero-Crossing** | Autocorrelation | 부호 반전 > 3회/4초 AND Autocorr Peak > 0.5 |
| **3800 (Coasting Down)** | **Decay Rate** | RMS Slope | log(Freq) 추세가 선형이며 RMS도 동반 하락 |

### 4. 추가 피처의 역할 및 구현 방안
 
 1. **dF/dt Slope (기울기):** 4초간의 Dominant_Freq 데이터 25개에 대해 선형 회귀를 돌려 a (Slope) 값을 산출합니다.
 
 2. **Cyclic 빈도 계산:** 4초간 dF/dt의 부호가 +에서 - 혹은 -에서 +로 바뀐 횟수를 카운트합니다.
 * **3410 (Cyclic Low):** 1~2회 (느린 부하 변동)
 * **3420 (Cyclic Mid):** 3~5회 (규칙적 반복 공정)
 * **3430 (Cyclic High):** 6회 이상 (고속 진동/Hunting)
 
 3. **Coasting Down 특화 로직:** 
 * 일반적인 **330 (Dynamic Dec)**은 인버터 제어로 인해 속도가 선형적으로 줄어들지만, **3800 (Coasting Down)**은 물리적 마찰에 의한 지수 감소입니다.
 * abs(Slope)가 시간에 따라 점점 줄어드는(Deceleration의 감속) 곡률을 체크하여 분리합니다.

### 💡 향후 구현 가이드 (정상 프로파일링 연계)
위 로직이 정교화되면, **정상 상태 학습**은 다음과 같이 진행됩니다.
 * 기기가 **3120 (Speed Mid)** 상태라고 판단하면, 해당 구간의 **평균 RMS와 Peak Amplitude**를 메모리에 기록합니다.
 * 만약 상태는 **3120**인데 기록된 평균보다 RMS가 20% 이상 높다면? → **attr.is_anomaly = 1**을 발생시킵니다.
 * 이로써 사용자는 "어떤 상태에서 왜 이상이 발생했는지"를 라벨링 없이도 즉시 알 수 있게 됩니다.

## III. 설비 상태 전이 히스테리시스 및 디바운스 설계
*** *** *** *** *** *** *** *** *** *** *** *** *** *** *** *** *** *** *** *** 

 설비 상태 인식의 신뢰도를 결정짓는 가장 핵심적인 **히스테리시스(Hysteresis)와 디바운스(Debounce) 정교화 설계**를 진행합니다.  
 진동 데이터는 기계적 마찰과 전기적 노이즈로 인해 경계값에서 매우 심하게 요동칩니다.  
 이를 제어하기 위해 **'공간적 여유(Threshold Margin)'**와 **'시간적 검증(Temporal Confirmation)'**을 결합한 이중 방어 체계를 설계합니다.

### 1. 상태 전이 디바운스 및 히스테리시스 마스터 기준

| 상태 전이 경로 (From -> To) | 판단 지표 | 디바운스 (Frames) | 히스테리시스 (Margin) | 설계 의도 |
|---|---|---|---|---|
| **10 (DOWN) -> 20 (IDLE)** | RMS 에너지 | 5 프레임 (0.8초) | + 20% (상향) | 미세 진동에 의한 오작동 방지 |
| **20 (IDLE) -> 10 (DOWN)** | RMS 에너지 | 20 프레임 (3.2초) | - 10% (하향) | 완전히 멈췄을 때만 DOWN 판정 |
| **20 (IDLE) -> 30 (RUN)** | Peak Amp / RMS | 10 프레임 (1.6초) | + 15% (상향) | 초기 구동 서지(Surge) 필터링 |
| **30 (RUN) -> 20 (IDLE)** | Peak Amp / RMS | 15 프레임 (2.4초) | - 15% (하향) | 작업 종료 후 잔여 진동 대기 |
| **310 (STEADY) <-> DYNAMIC (320, 330, 340)** | dF/dt Slope | 3 프레임 (0.48초) | Slope th * 1.2 | 속도 변화 즉각 감지 및 추적 |
| **NORMAL <-> ANOMALY** | mu + k*sigma | **1~2 프레임 (즉시)** | 이탈 시 마진 없음 | **안전 최우선: 즉시 트리거** |

### 2. 세부 상태별 '상태 확정' 알고리즘 (MECE 순차 필터)
  단순히 값이 넘었다고 상태를 바꾸지 않고, **'후보 상태(Candidate State)'**가 디바운스 카운트를 채워야 **'현재 상태(Active State)'**로 승격시키는 로직입니다.

#### 2.1. 에너지 계층 (L1) 디바운스 로직
 * **Active_RMS > RMS_High_Th:** IDLE_Confirm_Cnt 증가. 5회 도달 시 L1 = **20 (IDLE)**.
 * **Active_RMS < RMS_Low_Th:** DOWN_Confirm_Cnt 증가. 20회 도달 시 L1 = **10 (DOWN)**.
 * **효과:** 전동기 제어 전원 인가 시 발생하는 짧은 노이즈는 무시하고, 실제 안정적인 대기 상태만 포착합니다.

#### 2.2. 주파수 계층 (L2/L3) 히스테리시스 로직
 * **변화 감지:** 현재 Slope가 Stability_Th (0.5Hz/s)를 초과하면 즉시 동적 운전 Candidate(Acc/Dec)로 마킹.
 * **상태 래치(Latch):** 가속(320) 또는 감속(330) 등급이 한번 결정되면, Slope가 설정된 등급의 하한선보다 **10% 더 낮아질 때까지** 해당 등급을 유지.
 * **효과:** 가/감속 중 미세하게 속도가 주춤거려도 **3230 (Acc High)**에서 **3220 (Acc Mid)**로 왔다 갔다(Chattering) 하는 현상을 막아 데이터의 일관성을 유지합니다.

### 2.3. Dynamic RUN (동적 운전) 세부 판별 정교화
  가장 정밀한 제어가 필요한 **가/감속 및 반복(Cyclic)** 구간의 판별 로직입니다.

#### ① 가/감속 등급 전이 (Debounce 3)
 * 3프레임 연속으로 Slope가 특정 등급(High/Mid/Low) 영역에 머물러야 등급 변경을 허용합니다.
 * 기울기가 급격히 변할 경우(d2F/dt2 급증), 이전 카운트를 초기화하고 가장 최신의 기울기를 추적합니다.

#### ② 반복 패턴 (340 Cyclic) 확정 정책
 * **진입:** 4초(25프레임) 내에 Slope의 부호 반전이 3회 이상 발생하고, Autocorrelation Peak가 0.6을 넘으면 즉시 **340 (Dynamic Cyclic)** 계열로 강제 고정합니다.
 * **유지:** 패턴이 한번 확정되면, 이후 5초 동안은 Slope가 잠시 안정되어도 **310 (Steady RUN)**으로 돌아가지 않고 패턴 상태를 유지합니다. (Inching 공정 보호)

### 2.4. 이상 징후 (Anomaly) 트리거 및 홀드 타임
  Anomaly 속성은 히스테리시스보다 **'사후 보존(Hold)'**에 집중합니다.
 * **즉시 트리거:** 정상 범위를 벗어난 이상치가 **단 2프레임**만 감지되어도 즉시 is_anomaly = 1로 래치하고 기록을 시작합니다.
 * **기록 유지 (Hold Time):** 신호가 다시 정상 범위로 들어와도, **최소 5초(설정 가능)** 동안은 이상 상태를 유지합니다.
   * 이는 고장 직후 설비가 멈추는 과정(**3800 Coasting Down**)까지 모두 하나의 이벤트 파일에 담기 위함입니다.

### 3. 상태 판단 지표 및 연산 공식
  판단은 상위 레벨에서 하위 레벨로 내려가는 순차적 필터링 구조를 따릅니다.

#### 3.1 Step 1) 에너지 및 평탄도 (L1 판단)
 * **Total RMS:** 전체 주파수 대역의 에너지 실효치.
 * **Spectral Flatness:** 스펙트럼의 기하평균 / 산술평균.
   * 공식: Flatness = exp(sum(ln(bin_i)) / n) / (sum(bin_i) / n)
   * 해석: 1.0에 가까우면 백색 잡음, 0에 가까우면 순음(회전체).

#### 3.2 Step 2) 주파수 추세 분석 (L2/L3 가감속 판단)
 * **dF/dt Slope (기울기):** 윈도우 내 주파수 피크값들에 대한 선형 회귀 기울기.
   * 공식: Slope = (n * sum(t*F) - sum(t) * sum(F)) / (n * sum(t^2) - (sum(t))^2)
 * **R-Square (선형성):** 기울기 추정의 신뢰도.
   * 공식: R2 = 1 - (sum((F_obs - F_pred)^2) / sum((F_obs - F_mean)^2))
   * 해석: 0.7 이상이면 선형 가/감속(Acc/Dec), 낮으면 불규칙(Random/Cyclic).

#### 3.3 Step 3) 주기성 분석 (Cyclic 판단)
 * **Autocorrelation (자기상관):** Delta F 신호의 주기성 확인.
   * 공식: ACF(k) = sum((X_i - mean) * (X_{i+k} - mean)) / Variance
   * 해석: 지연 시점(k)에서 피크 발생 시 해당 주기를 Cyclic 빈도로 확정.

### 1. 전처리 설계: 데이터 수집 및 합성 모드

### 3. [보완] 과도 상태(Transient) 특화 예외 처리 및 히스테리시스
가속 및 감속 구간에서 발생하는 구조적 진동 상승을 '이상'으로 오판하지 않도록 감도를 자동 조절합니다.
#### 3.1. 과도 상태 Anomaly 감도 스케일링
 * **Steady (310) 상태:** Threshold = mu + k * sigma (일반 감도)
 * **Transient (320, 330) 상태:** Threshold = (mu + k * sigma) * Transient_K_Factor
   * Transient_K_Factor: 기본값 **1.5** (가감속 중에는 1.5배 둔감하게 설정)
 * **효과:** 모터 기동/정지 시 발생하는 정상적인 진동 서지에 의한 오알람 방지.
#### 3.2. 상태 전이 디바운스 및 히스테리시스 요약
| 상태 전이 경로 | 디바운스 (연속 프레임) | 히스테리시스 마진 | 비고 |
|---|---|---|---|
| **IDLE ➔ RUN** | 10 프레임 (1.6s) | + 15% | 초기 서지 안정화 대기 |
| **STEADY ↔ DYNAMIC** | 3 프레임 (0.48s) | Slope Th * 1.2 | 속도 변화 즉시 추적 |
| **NORMAL ↔ ANOMALY** | **2 프레임 (즉시)** | 없음 (즉시 대응) | **과도 상태 시 K-Factor 적용** |
