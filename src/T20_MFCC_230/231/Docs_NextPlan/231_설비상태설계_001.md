T20 시스템의 설비 상태 인식(Machine State Awareness)을 고도화하기 위한 **세부 상태 정의와 판단 로직 설계**를 진행하겠습니다.
단순한 ON/OFF를 넘어, 설비의 물리적 동작 특성(RPM, 부하량, 전이 속도)을 피쳐 도메인에서 구분함으로써 **"정상적인 변화"와 "이상 진동"을 완벽히 분리**하는 것이 이번 설계의 핵심입니다.
### 1. 설비 상태 모드(States) 세분화 및 판단 방법
#### [A] OFF (Power Off)
 * **정의:** 설비에 전원이 완전히 차단된 상태.
 * **판단 방법:** * Total RMS < Sensor Floor Noise (사전에 학습된 센서 자체 노이즈 하한선).
   * Spectral Flatness ≈ 1.0 (특정 주파수 집중 없이 백색 잡음만 존재).
 * **비고:** 이 상태에서는 모든 트리거 감시를 중단하여 전력을 극단적으로 보존합니다.
#### [B] IDLE (Standby & Ready)
 * **추가 구분:**
   1. **IDLE_QUIET:** 제어 전원만 인가됨. 냉각 팬 등 미세 진동도 없는 상태.
   2. **IDLE_ACTIVE:** 메인 모터는 정지했으나 냉각 팬, 유압 펌프 등 보조 장치가 가동 중인 상태.
 * **판단 방법:**
   * Floor Noise < Total RMS < Run Threshold.
   * Top-1 Peak Frequency가 전원 주파수(60Hz) 또는 팬 회전수에 고정됨.
   * **IDLE_ACTIVE**는 특정 저주파 대역에 약한 에너지가 지속 관찰됨.
#### [C] RUN_STABLE (Steady State)
 * **정의:** 설정된 목표 속도로 정속 주행 중인 상태.
 * **판단 방법:** |ΔF (Top-1 Freq)| < 안정 한계치 (주파수 변동이 거의 없음).
 * **세부 등급 구분:**
   1. **Speed Grading:** Top-1 Peak의 주파수를 RPM으로 환산하여 등급화 (예: Low < 30Hz, Mid 30~50Hz, High > 50Hz).
   2. **Load vs. No-Load:** * **No-Load (무부하):** 정속 주행 중 Top-1 Amp 및 Total RMS가 기준 프로파일의 하한선에 위치.
     * **Loaded (부하):** 동일 RPM에서 무부하 대비 Total RMS가 유의미하게 상승하고, Harmonic Energy Ratio(고조파 에너지 비율)가 증가함.
#### [D] RUN_TRANSIENT (Transition State)
 * **정의:** 속도가 변하거나 부하가 급변하는 구간.
 * **판단 방법:** |ΔF (Top-1 Freq)| > 전이 한계치 또는 Spectral Flux 폭증.
 * **세부 구분:**
   1. **Acceleration (가속):** ΔF > 0이 일정 프레임 유지.
   2. **Deceleration (감속):** ΔF < 0이 일정 프레임 유지.
   3. **Cyclic/Pattern (반복):** 가/감속이 짧은 주기로 반복됨 (Inching 또는 간헐적 작업).
   4. **Transition Grade:** |ΔF|의 기울기(Slope)로 급가속/완만가속 구분.
   5. **Load During Transition:** Δ(Peak Amplitude)의 변화율을 통해 과부하 상태에서 가속 중인지 판단.
#### [E] 추가 검토 상태
 * **COASTING DOWN:** 전원은 차단되었으나 관성에 의해 회전하며 멈추는 상태 (감속과 유사하나 고유의 지수적 감쇠 패턴 존재).
 * **RESONANCE (공진):** 특정 RPM 등급에서만 진폭이 비정상적으로 커지는 구간 (고장으로 오판하기 쉬우므로 별도 프로파일링 필요).
### 2. 설비 상태 판단 로직트리 (Logic Tree)
설비의 상태를 결정하기 위한 순차적 의사결정 구조입니다. 이 로직은 T231_Dsp_Pipeline에서 매 프레임(1600Hz 기준 256샘플마다) 수행됩니다.
 1. **Level 1: 에너지 유무 검사**
   * Total\_RMS < Floor\_Noise ?
     * YES → **[OFF]**
     * NO → Go to Level 2
 2. **Level 2: 회전 성분 검사**
   * Spectral\_Flatness > 0.8 (매우 평탄) OR Top1\_Amp < Run\_Threshold ?
     * YES → **[IDLE]** * 보조 주파수 성분 존재? YES → **[IDLE_ACTIVE]** / NO → **[IDLE_QUIET]**
     * NO → Go to Level 3 (가동 중 확정)
 3. **Level 3: 속도 안정성 검사 (\Delta F)**
   * |\Delta F (Top1)| < \epsilon (안정) ?
     * YES → **[RUN_STABLE]**
       * RMS vs Baseline(\mu, \sigma) 비교 → **[Loaded]** or **[No-Load]**
       * Freq 값에 따른 등급 부여 → **[Low / Mid / High Speed]**
     * NO → **[RUN_TRANSIENT]**
       * \Delta F > 0 → **[Acceleration]** (기울기에 따라 Grade 부여)
       * \Delta F < 0 → **[Deceleration]**
       * 부하 지표 병행 확인
 4. **Level 4: 프로파일 이탈 검사 (Final Filter)**
   * 위 상태 중 어디에 해당하더라도, 해당 상태의 **적응형 임계값(\mu + k\sigma)**을 초과하는 피쳐가 발생했는가?
     * YES → **[ANOMALY]** 판정 및 이벤트 로그 생성
### 3. 향후 설계 방향 (Hysteresis 연계)
상기 상태 모드 구분이 완료되면, **트리거 히스테리시스**는 다음과 같이 더욱 강력해집니다.
 * **상태별 히스테리시스 차등화:**
   * **IDLE → RUN 전이 시:** 기동 초기 충격을 오알람으로 잡지 않도록 **디바운스 카운터**를 길게 설정.
   * **RUN_STABLE 상태 시:** 아주 미세한 이상도 포착해야 하므로 **히스테리시스 밴드**를 좁게 설정하여 감도 극대화.
   * **TRANSIENT 상태 시:** 가감속 중의 진동 폭증은 정상이므로, **임계값을 일시적으로 상향(Masking)**하거나 트리거를 유예.
이 로직트리를 바탕으로 T231_Dsp_Pipeline의 피쳐 추출부와 T230_Mfcc_Cor의 상태 전이 엔진을 업데이트할 준비가 되었습니다. 우선 **"Level 3까지의 상태 판별 알고리즘"** 구현을 위해 코드 작성을 시작할까요?

# 추가 설계 메모

## 히스테리시스 공통 전략
- Enter Threshold ≠ Exit Threshold 구조 적용
- 상태 전이 조건:
  - Enter: Feature > Threshold_high
  - Exit: Feature < Threshold_low
- Threshold_high = 기준값 + delta
- Threshold_low = 기준값 - delta

## 디바운스 공통 전략
- 상태 전이는 반드시 "연속 프레임 조건" 만족 필요
- 기본 구조:
  - if condition == TRUE for N frames → 상태 변경
- 상태별 권장 프레임:
  - OFF/IDLE: 10~20
  - RUN 진입: 20~30
  - TRANSIENT: 5~10

## TRANSIENT 구간 보호 (핵심)
- masking_factor 적용:
  - effective_threshold = base_threshold * (1 + beta)
- 또는 anomaly detection 일시 hold

## 최종 구조 요약
1. 상태 판별 (Logic Tree)
2. 상태별 히스테리시스 적용
3. 디바운스 필터 통과
4. 이상 탐지 (상태 기반 adaptive threshold)

### 2. 설비 상태 관련 구조체 초안 설계 v2(C++)

```cpp
#pragma once

#include <stdint.h>

/**
 * @brief 설비 가동 상태 대분류 (Large Category)
 */
enum EM_T20_MachineState {
    EN_T20_STATE_OFF = 0,           // 전원 차단
    EN_T20_STATE_IDLE,              // 대기/준비
    EN_T20_STATE_RUN_STABLE,        // 정속 가동
    EN_T20_STATE_RUN_TRANSIENT,     // 과도 상태 (가감속)
    EN_T20_STATE_ANOMALY            // 이상 발생
};

/**
 * @brief 설비 상태 소분류 (Small Category)
 */
enum EM_T20_SubState {
    EN_T20_SUB_NONE = 0,
    // IDLE
    EN_T20_SUB_IDLE_QUIET,          // 정적 대기
    EN_T20_SUB_IDLE_ACTIVE,         // 보조기기 가동 중
    // RUN_STABLE
    EN_T20_SUB_STABLE_NOLOAD,       // 정속 무부하
    EN_T20_SUB_STABLE_LOADED,       // 정속 부하
    // RUN_TRANSIENT (세분화됨)
    EN_T20_SUB_TRANS_ACCEL,         // 가속 구간
    EN_T20_SUB_TRANS_DECEL,         // 감속 구간
    EN_T20_SUB_TRANS_CYCLIC         // 가/감속 반복 (Inching)
};

/**
 * @brief 설비 상태 인식 및 히스테리시스 관리 컨텍스트
 */
struct ST_T20_MachineStateCtx_t {
    // [1. 현재 상태]
    EM_T20_MachineState group;      // 대분류
    EM_T20_SubState     sub;        // 소분류

    // [2. 판단 지표]
    float rms_now;                  // 현재 실효치
    float freq_peak;                // 메인 주파수 피크 (Hz)
    float freq_delta;               // 이전 프레임 대비 주파수 변화량
    float flatness;                 // 스펙트럼 평탄도
    float peak_hold_max;            // 충격 탐지용 최대 절대값

    // [3. 디바운싱 카운터]
    uint16_t confirm_cnt;           // 상태 확정용 연속 카운터
    uint16_t transition_cnt;        // 상태 전이 대기 카운터
    uint16_t sign_flip_cnt;         // 주파수 변화 부호 반전 횟수 (반복 판별용)

    // [4. 히스테리시스 및 트리거 전략]
    bool     is_latched;            // 트리거 실행 중 여부
    uint32_t hold_timer_ms;         // 트리거 해제 유예 시간
    float    active_threshold;      // 현재 상태 가변 임계값 (mu + k*sigma)
    float    margin_ratio;          // 전이 방지 마진 (기본 0.15)

    ST_T20_MachineStateCtx_t() {
        group = EN_T20_STATE_OFF;
        sub = EN_T20_SUB_NONE;
        rms_now = 0.0f;
        freq_peak = 0.0f;
        freq_delta = 0.0f;
        flatness = 1.0f;
        peak_hold_max = 0.0f;
        confirm_cnt = 0;
        transition_cnt = 0;
        sign_flip_cnt = 0;
        is_latched = false;
        hold_timer_ms = 0;
        active_threshold = 0.0f;
        margin_ratio = 0.15f;
    }
};

    float    adaptive_threshold;    // 현재 상태(State)에 맞춰 스위칭된 가변 임계값
    float    hysteresis_margin;     // 상태 이탈을 막기 위한 여유분 (예: 0.15)

        transition_frames_cnt = 0;
        cyclic_pattern_cnt = 0;

        is_latched = false;
        latch_hold_timer_ms = 0;
        adaptive_threshold = 0.0f;
        hysteresis_margin = 0.15f; // 15% 마진 기본값
    }
};
