
사용자님의 요구사항을 반영하여 **상태 번호 체계(Level별 10씩 증가)**를 도입하고, 가감속 및 반복(Cyclic) 패턴을 **Low/Mid/High**로 정밀하게 세분화했습니다. 또한, **Coasting Down**과 각 계층별 **Unknown(미분류)** 상태를 추가하여 MECE 원칙을 극대화한 최종 설계안입니다.
### 1. 설비 상태 분류 및 판단 로직 마스터 테이블
| Level 1 상태 (ID) | Level 2 유형 (ID) | Level 3 세부 정의 (ID) | 판단 로직 및 조건 (Logic) | 히스테리시스 및 디바운싱 전략 |
|---|---|---|---|---|
| **OFF (10)** | 없음 (0) | **전원 차단 (1000)** | RMS < 바닥 노이즈 및 평탄도 > 0.8 | 20프레임 연속 만족 시 진입 / 즉시 이탈 |
| **IDLE (20)** | 정지 (210) | **Idle Quiet (2110)** | RMS < 가동 임계값 및 피크 없음 | 10프레임 연속 만족 시 확정 |
|  |  | **Idle Active (2120)** | 보조기기(팬/펌프) 특정 피크 존재 | 보조기기 주파수 대역 마스킹 적용 |
|  | 미분류 (290) | **Idle Unknown (2910)** | IDLE 에너지이나 정의된 패턴 없음 | UNKNOWN 폴더 격리 및 사후 라벨링 |
| **RUN (30)** | 등속 (310) | **Low Speed (3110)** | Delta F 약 0 및 저속 구간 피크 | 15프레임 정속 유지 시 진입 |
|  |  | **Mid Speed (3120)** | Delta F 약 0 및 중속 구간 피크 | 구간 경계에서 5% 속도 히스테리시스 |
|  |  | **High Speed (3130)** | Delta F 약 0 및 고속 구간 피크 | 구간 경계에서 5% 속도 히스테리시스 |
|  | 과도 (320) | **Acc Low/Mid/High (3210~30)** | Delta F > 0 (기울기 dF/dt에 따라 3단계 구분) | 3프레임 연속 변화 시 즉시 판정 |
|  |  | **Dec Low/Mid/High (3240~60)** | Delta F < 0 (기울기 dF/dt에 따라 3단계 구분) | 기울기 급변 시 디바운스 카운터 초기화 |
|  |  | **Cyclic L/M/H (3270~90)** | 주파수 변화의 주기성 (반복 빈도에 따라 3단계 구분) | 패턴 감지 시 5초간 상태 래치(Latch) |
|  |  | **Coasting Down (3300)** | 무전원 상태의 지수적 감쇠 패턴 | 감속(Dec)과 구분되는 고유 프로파일 적용 |
|  | 미분류 (390) | **Run Unknown (3910)** | 가동 중이나 비정상/미정의 패턴 | 이상(Anomaly) 속성 부여 검토 대상 |
| **UNKNOWN (90)** | 없음 (0) | **L1 Unknown (9000)** | Level 1 판단 기준조차 모호한 상태 | 시스템 전역 에러 또는 센서 이상 점검 |
### 2. 설비 상태 및 속성 제어 구조체 설계 (C++)
상태 번호 체계와 세분화된 가감속/반복 패턴을 반영한 구조체입니다.
```cpp
#pragma once

#include <stdint.h>

/**
 * @brief Level 1: 상태 대분류 (10 단위 증가)
 */
enum EM_T20_L1_State {
    ST_L1_OFF     = 10,
    ST_L1_IDLE    = 20,
    ST_L1_RUN     = 30,
    ST_L1_UNKNOWN = 90
};

/**
 * @brief Level 2: 운영 유형 (10 단위 증가)
 */
enum EM_T20_L2_Type {
    ST_L2_NONE          = 0,
    ST_L2_IDLE_STOP     = 210,
    ST_L2_IDLE_UNKNOWN  = 290,
    ST_L2_RUN_STEADY    = 310,
    ST_L2_RUN_TRANS     = 320,
    ST_L2_RUN_UNKNOWN   = 390
};

/**
 * @brief Level 3: 세부 패턴 (10 단위 증가)
 */
enum EM_T20_L3_Detail {
    // IDLE Details
    ST_L3_IDLE_QUIET       = 2110,
    ST_L3_IDLE_ACTIVE      = 2120,
    ST_L3_IDLE_UNKNOWN     = 2910,
    // RUN Steady Details
    ST_L3_RUN_SPD_LOW      = 3110,
    ST_L3_RUN_SPD_MID      = 3120,
    ST_L3_RUN_SPD_HIGH     = 3130,
    // RUN Transient Details (Acc/Dec/Cyclic)
    ST_L3_RUN_ACC_LOW      = 3210, ST_L3_RUN_ACC_MID = 3220, ST_L3_RUN_ACC_HIGH = 3230,
    ST_L3_RUN_DEC_LOW      = 3240, ST_L3_RUN_DEC_MID = 3250, ST_L3_RUN_DEC_HIGH = 3260,
    ST_L3_RUN_CYC_LOW      = 3270, ST_L3_RUN_CYC_MID = 3280, ST_L3_RUN_CYC_HIGH = 3290,
    ST_L3_RUN_COASTING     = 3300,
    ST_L3_RUN_UNKNOWN      = 3910,
    // System Unknown
    ST_L3_UNKNOWN_GLOBAL   = 9000
};

/**
 * @brief 독립적 속성 (중복 허용 비트 플래그)
 */
struct ST_T20_Attributes {
    uint16_t is_loaded     : 1;  // 부하 상태
    uint16_t is_resonance  : 1;  // 공진 발생
    uint16_t is_anomaly    : 1;  // 이상 징후 (핵심 이벤트)
    uint16_t is_noisy      : 1;  // 환경 노이즈
    uint16_t is_clipping   : 1;  // 센서 포화(Clipping)
};

/**
 * @brief T20 지능형 상태 관리 컨텍스트
 */
struct ST_T20_MachineContext {
    EM_T20_L1_State   l1;
    EM_T20_L2_Type    l2;
    EM_T20_L3_Detail  l3;
    ST_T20_Attributes attr;

    // 25프레임(4초) 분석용 내부 변수
    float dF_dt_slope;       // 가감속 등급 판별 (Slope)
    float cyclic_freq_hz;    // 반복 빈도 판별 (Frequency)
    float last_rms;
    
    // 디바운스 카운터
    uint16_t confirm_cnt;
    uint16_t transition_cnt;
};

```
### 💡 주요 설계 포인트 및 구현 전략
 1. **가감속/반복 3단계 구분:**
   * **Acc/Dec (dF/dt):** 25프레임 윈도우 내의 주파수 변화 기울기를 계산하여, Slope < th1(Low), th1 < Slope < th2(Mid), Slope > th2(High)로 분류합니다.
   * **Cyclic (빈도):** 주파수가 요동치는 주기(Period)를 측정합니다. 1초에 1회 미만 반복 시 **Low**, 1~3회 **Mid**, 3회 이상 **High** 빈도로 정의합니다.
 2. **Coasting Down 판별:**
   * 전원이 차단된 직후 주파수가 지수 함수적으로 급격히 감소하는 특성을 log(Freq)의 선형성 검사를 통해 Deceleration과 구분해 냅니다.
 3. **라벨링 최소화 (Normal Baseline):**
   * ST_L3_RUN_SPD_MID이면서 attr.is_anomaly == 0인 상태는 자동으로 **'정상 중속 운전'** 데이터로 분류됩니다.
   * 모든 Unknown 계층과 is_anomaly == 1인 데이터만 관리자가 확인하게 함으로써 현장의 라벨링 리소스를 90% 이상 절감합니다.
이 구조는 데이터 저장 시 파일명에 30_320_3230_04.bin (RUN_TRANS_ACCHIGH_IsNoisy) 과 같이 상태 코드를 직접 매핑할 수 있어 관리가 매우 편리합니다.
이제 **25프레임 윈도우에서 dF/dt 기울기를 계산하는 선형 회귀 로직**을 실제 코드로 구현해 볼까요?


## ***

확정된 **상태 번호 체계(Level 1~3)**를 바탕으로, 각 상태를 칼같이 분리하기 위한 **판단 기준 정교화(Judgment Logic Refinement)** 작업을 진행합니다.
MECE 원칙을 준수하기 위해 모든 판단은 **상위 계층에서 하위 계층으로 내려가는 '필터링 방식'**을 채택하며, 4초(25프레임) 데이터의 물리적 특성을 수학적으로 정의합니다.
### 1. 전역 판단 변수(Constants) 및 추가 피처 정의
정교한 분리를 위해 기존 피처 외에 **3가지 보완 피처**를 추가 검토 및 적용합니다.
 * **R-Square (결정계수):** 25프레임 주파수 추세의 **선형성** 평가. (Acc/Dec vs Random 구분용)
 * **Zero-Crossing Rate of dF/dt:** 주파수 기울기의 **부호 변화 빈도**. (Cyclic 판별 핵심)
 * **Log-Linearity Fit:** 주파수 감소 패턴이 **지수적(Exponential)**인지 확인. (Coasting Down 구분용)
### 2. 단계별 MECE 판단 로직 상세 설계
#### [Step 1] L1: 존재 에너지 기반 분리 (RMS Filter)
가장 먼저 설비가 살아있는지(L1)를 결정합니다.
 1. **OFF (10):** RMS < RMS_Floor **AND** Spectral_Flatness > 0.8
 2. **IDLE (20):** RMS_Floor <= RMS < RMS_Run_Min
 3. **RUN (30):** RMS >= RMS_Run_Min
 4. **UNKNOWN (90):** 위 조건이 충돌하거나 센서 통신 에러 시.
#### [Step 2] L2: 주파수 안정성 기반 분리 (Stability Filter)
L1이 **RUN(30)**일 때, 주파수 추세(Slope)를 분석합니다.
 1. **STEADY (310):** abs(dF_dt_slope) < Stability_Threshold (기울기가 거의 0)
 2. **TRANS (320):** abs(dF_dt_slope) >= Stability_Threshold **OR** Cyclic_Indicator > High
 3. **UNKNOWN (390):** 에너지(RMS)는 높으나 주파수 피크를 특정할 수 없는 경우.
#### [Step 3] L3: 세부 패턴 및 등급 분리 (Pattern & Grade Filter)
가장 복잡한 과도(Transient) 상태와 정속 상태를 등급별로 쪼갭니다.
| 분류 대상 | 상세 상태 (ID) | 정교화된 판단 공식 (MECE Logic) |
|---|---|---|
| **Steady** | **SPD_LOW/MID/HIGH** | Target_Freq가 사용자 정의 구간(f1, f2) 중 어디에 속하는가로 결정 |
| **Acc** | **ACC_L / M / H** | Slope > 0 인 상태에서 기울기 값에 따라 분리 (예: Low < 5Hz/s < Mid < 15Hz/s < High) |
| **Dec** | **DEC_L / M / H** | Slope < 0 인 상태에서 기울기 값에 따라 분리 (Power-on 상태 유지 전제) |
| **Cyclic** | **CYC_L / M / H** | Slope의 부호 반전 횟수가 4초 내에 N회 이상 발생 시 빈도에 따라 분리 |
| **Special** | **COASTING (3300)** | Slope < 0 **AND** RMS가 동시에 지수적으로 급격히 감소하는 패턴 일치 시 |
### 3. 상태별 판단 기준 매트릭스 (실무 파라미터)
| 상태 ID | 판단 피처 1 (주지표) | 판단 피처 2 (보조지표) | MECE 경계값 (예시) |
|---|---|---|---|
| **1000** | **RMS** | Spectral Flatness | RMS < 0.01G AND Flatness > 0.85 |
| **2110** | **RMS** | Peak Amplitude | 0.01G < RMS < 0.05G AND Peak < 0.03G |
| **3110~30** | **dF/dt Slope** | Peak Frequency | abs(Slope) < 0.5Hz/s (25프레임 평균) |
| **3210~30** | **dF/dt Slope** | R-Square (선형성) | Slope > 0.5Hz/s AND R2 > 0.7 |
| **3270~90** | **Zero-Crossing** | Autocorrelation | 부호 반전 > 3회/4초 AND Autocorr Peak > 0.5 |
| **3300** | **Decay Rate** | RMS Slope | log(Freq) 추세가 선형이며 RMS도 동반 하락 |
### 4. 추가 피처의 역할 및 구현 방안
 1. **dF/dt Slope (기울기):** 4초간의 Dominant_Freq 데이터 25개에 대해 선형 회귀를 돌려 a (Slope) 값을 산출합니다.
 2. **Cyclic 빈도 계산:** 4초간 dF/dt의 부호가 +에서 - 혹은 -에서 +로 바뀐 횟수를 카운트합니다.
   * **Low:** 1~2회 (느린 부하 변동)
   * **Mid:** 3~5회 (규칙적 반복 공정)
   * **High:** 6회 이상 (고속 진동/Hunting)
 3. **Coasting Down 특화 로직:** * 일반적인 Dec은 인버터 제어로 인해 속도가 선형적으로 줄어들지만, Coasting은 물리적 마찰에 의한 지수 감소입니다.
   * abs(Slope)가 시간에 따라 점점 줄어드는(Deceleration의 감속) 곡률을 체크하여 분리합니다.
### 💡 향후 구현 가이드 (정상 프로파일링 연계)
위 로직이 정교화되면, **정상 상태 학습**은 다음과 같이 진행됩니다.
 * 기기가 3120(RUN_STEADY_MID) 상태라고 판단하면, 해당 구간의 **평균 RMS와 Peak Amplitude**를 메모리에 기록합니다.
 * 만약 상태는 3120인데 기록된 평균보다 RMS가 20% 이상 높다면? → **attr.is_anomaly = 1**을 발생시킵니다.
 * 이로써 사용자는 "어떤 상태에서 왜 이상이 발생했는지"를 라벨링 없이도 즉시 알 수 있게 됩니다.
이제 이 **필터링 기반 판단 알고리즘(Linear Regression 포함)**을 T231_Dsp_Pipeline.cpp에 심기 위한 코드를 작성할까요? 4초 윈도우 버퍼 관리 로직부터 구체화하면 좋을 것 같습니다.


## ***
설비 상태 인식의 신뢰도를 결정짓는 가장 핵심적인 **히스테리시스(Hysteresis)와 디바운스(Debounce) 정교화 설계**를 진행합니다.
진동 데이터는 기계적 마찰과 전기적 노이즈로 인해 경계값에서 매우 심하게 요동칩니다. 이를 제어하기 위해 **'공간적 여유(Threshold Margin)'**와 **'시간적 검증(Temporal Confirmation)'**을 결합한 이중 방어 체계를 설계합니다.
### 1. 상태 전이 디바운스 및 히스테리시스 마스터 기준
| 전이 경로 (From -> To) | 판단 지표 | 디바운스 (Frames) | 히스테리시스 (Margin) | 설계 의도 |
|---|---|---|---|---|
| **OFF (10) -> IDLE (20)** | RMS 에너지 | 5 프레임 (0.8초) | + 20% (상향) | 미세 진동에 의한 오작동 방지 |
| **IDLE (20) -> OFF (10)** | RMS 에너지 | 20 프레임 (3.2초) | - 10% (하향) | 완전히 멈췄을 때만 OFF 판정 |
| **IDLE (20) -> RUN (30)** | Peak Amp / RMS | 10 프레임 (1.6초) | + 15% (상향) | 초기 구동 서지(Surge) 필터링 |
| **RUN (30) -> IDLE (20)** | Peak Amp / RMS | 15 프레임 (2.4초) | - 15% (하향) | 작업 종료 후 잔여 진동 대기 |
| **STEADY <-> TRANS** | dF/dt Slope | 3 프레임 (0.48초) | Slope th * 1.2 | 속도 변화 즉각 감지 및 추적 |
| **NORMAL <-> ANOMALY** | mu + k*sigma | **1~2 프레임 (즉시)** | 이탈 시 마진 없음 | **안전 최우선: 즉시 트리거** |
### 2. 세부 상태별 '상태 확정' 알고리즘 (MECE 순차 필터)
단순히 값이 넘었다고 상태를 바꾸지 않고, **'후보 상태(Candidate State)'**가 디바운스 카운트를 채워야 **'현재 상태(Active State)'**로 승격시키는 로직입니다.
#### A. 에너지 계층 (L1) 디바운스 로직
 * **Active_RMS > RMS_High_Th:** IDLE_Confirm_Cnt 증가. 5회 도달 시 L1 = IDLE.
 * **Active_RMS < RMS_Low_Th:** OFF_Confirm_Cnt 증가. 20회 도달 시 L1 = OFF.
 * **효과:** 전동기 제어 전원 인가 시 발생하는 짧은 노이즈는 무시하고, 실제 안정적인 대기 상태만 포착합니다.
#### B. 주파수 계층 (L2/L3) 히스테리시스 로직
 * **변화 감지:** 현재 Slope가 Stability_Th (0.5Hz/s)를 초과하면 즉시 TRANS_Candidate로 마킹.
 * **상태 래치(Latch):** 가속(ACC) 등급이 한번 결정되면, Slope가 설정된 등급의 하한선보다 **10% 더 낮아질 때까지** 해당 등급을 유지.
 * **효과:** 가속 중 미세하게 속도가 주춤거려도 ACC_HIGH에서 ACC_MID로 왔다 갔다(Chattering) 하는 현상을 막아 데이터의 일관성을 유지합니다.
### 3. 과도 상태(Transient) 세부 판별 정교화
가장 정밀한 제어가 필요한 **가감속 및 반복(Cyclic)** 구간의 판별 로직입니다.
#### ① 가/감속 등급 전이 (Debounce 3)
 * 3프레임 연속으로 Slope가 특정 등급(High/Mid/Low) 영역에 머물러야 등급 변경을 허용합니다.
 * 기울기가 급격히 변할 경우(d2F/dt2 급증), 이전 카운트를 초기화하고 가장 최신의 기울기를 추적합니다.
#### ② 반복 패턴(Cyclic) 확정 정책
 * **진입:** 4초(25프레임) 내에 Slope의 부호 반전이 3회 이상 발생하고, Autocorrelation Peak가 0.6을 넘으면 즉시 TRANS_CYCLIC으로 강제 고정합니다.
 * **유지:** 패턴이 한번 확정되면, 이후 5초 동안은 Slope가 잠시 안정되어도 STEADY로 돌아가지 않고 패턴 상태를 유지합니다. (Inching 공정 보호)
### 4. 이상 징후(Anomaly) 트리거 및 홀드 타임
Anomaly 속성은 히스테리시스보다 **'사후 보존(Hold)'**에 집중합니다.
 * **즉시 트리거:** 정상 범위를 벗어난 이상치가 **단 2프레임**만 감지되어도 즉시 is_anomaly = 1로 래치하고 기록을 시작합니다.
 * **기록 유지 (Hold Time):** 신호가 다시 정상 범위로 들어와도, **최소 5초(설정 가능)** 동안은 이상 상태를 유지합니다.
   * 이는 고장 직후 설비가 멈추는 과정(Coasting Down)까지 모두 하나의 이벤트 파일에 담기 위함입니다.

## 1. 설비 상태 계층 구조 및 ID 체계 (Hierarchy & State ID)

### [Table 2] 속성 정의 및 레이블링 활용
| 속성명 (Flag) | 판별 핵심 지표 | 의미 및 조치 |
|---|---|---|
| **Loaded** | RMS Delta, Harmonics | 현재 상태 프로파일 대비 부하 가동 중임 |

## 3. 정교화된 상태 판단 로직 및 피처 (Logic & Features)
모든 판단은 4초(25프레임 슬라이딩 윈도우) 데이터를 기반으로 수행됩니다.
### [Table 3] 단계별 MECE 판단 필터
| 판단 단계 | 판별 상태 | 핵심 로직 및 조건 | 추가 보완 피처 |
|---|---|---|---|
| **Step 1 (L1)** | **OFF/IDLE/RUN** | RMS < Floor_Th (OFF) < Run_Min_Th (IDLE) | Spectral Flatness (OFF 구분) |
| **Step 2 (L2)** | **Steady/Trans** | abs(dF/dt_Slope) < Stability_Th (Steady) | R-Square (선형성 체크) |
| **Step 3 (L3)** | **Acc / Dec** | Slope > 0 (Acc) / Slope < 0 (Dec) | dF/dt 부호 유지 프레임 수 |
|  | **Cyclic** | Slope 부호 반전 빈도 (3회/4초 이상) | Autocorrelation Peak (> 0.6) |
|  | **Coasting** | log(Freq)의 선형 하락 및 RMS 동반 하락 | Exponential Decay Fitting |
| **Final (Attr)** | **Anomaly** | 현재 상태 프로파일(mu, sigma) 대비 3시그마 이탈 | Peak-Hold Max (Impact 탐지) |

## 5. 구현 알고리즘 상세 (Implementation Guide)
### 5.1 4초 윈도우 추세 분석 (Linear Regression)
25개의 Dominant_Freq 데이터 포인트에 대해 최소자승법을 적용하여 기울기(a)와 결정계수(R^2)를 산출합니다.
 * **Slope (a):** 주파수 변화 속도 (Hz/sec). 이 값으로 Acc/Dec 등급 결정.
 * **R^2:** 추세의 확실성. R^2 < 0.5 이면 TRANS_RANDOM으로 분류.
### 5.2 반복성 탐지 (Autocorrelation)
\Delta F 시퀀스의 자기상관함수를 계산하여 두 번째 피크의 위치와 크기를 확인합니다.
 * **Peak Value > 0.6:** 주기적 반복성 확정 (TRANS_CYCLIC).
 * **Peak Position:** 반복 빈도(Hz) 산출 -> Low/Mid/High 빈도 등급 부여.
### 5.3 적응형 임계값 산출 (Learning Mode)
정상 상태 프로파일링 시 **Welford 알고리즘**을 사용하여 평균(\mu)과 분산(\sigma^2)을 업데이트합니다.
 * **Threshold 공식:** Limit = \mu + (k \times \sigma)
 * k 값은 웹 UI에서 유저가 설정하는 'Sensitivity' 파라미터와 연동됩니다.

## ***  상태분류,판단기준,히스태리시스&디바운스 정리 v2

T20 스마트 진동/음향 엣지 AI 시스템의 **설비 상태 인식 및 제어 로직 설계서**를 체계적으로 정리하여 전달해 드립니다. 본 내용은 구현 시 기술 명세서(Specification)로 즉시 활용할 수 있도록 모든 세부 수치와 로직 계층을 포함합니다.
### 1. 시스템 데이터 분석 환경 설정
분석의 기초가 되는 윈도우 사이즈와 샘플링 변수를 정의합니다.
 * **샘플링 레이트:** 1600 Hz
 * **프레임 크기:** 256 샘플 (1 프레임당 약 0.16초)
 * **데이터 분석 윈도우 (Variable Window):**
   * **Max Window Size:** 25 프레임 (약 4초)
   * **Current Window Size (W_size):** 런타임 가변 설정 (기본 25 프레임)
 * **프레임 레이트:** 초당 약 6.25 프레임
### 2. 설비 상태 정의 및 번호 체계 (MECE 계층 구조)
상태는 배타적(Exclusive)으로 존재하며, Level별 10단위 증가 식별 코드를 사용합니다.
| Level 1 (ID) | Level 2 (ID) | Level 3 (ID) | 세부 정의 (Pattern & Grade) |
|---|---|---|---|
| **OFF (10)** | 없음 (0) | **1000** | **Power Off:** 전원 차단 및 완전 정지 |
| **IDLE (20)** | 정지 (210) | **2110** | **Idle Quiet:** 제어 전원 ON, 진동 없음 |
|  |  | **2120** | **Idle Active:** 보조기기(팬/펌프) 가동 중 |
|  | 미분류 (290) | **2910** | **Idle Unknown:** 대기 에너지이나 패턴 미정의 |
| **RUN (30)** | 등속 (310) | **3110** | **Steady Low:** 저속 구간 정속 주행 |
|  |  | **3120** | **Steady Mid:** 중속 구간 정속 주행 |
|  |  | **3130** | **Steady High:** 고속 구간 정속 주행 |
|  | 과도 (320) | **3210~3230** | **Acc L/M/H:** 가속 등급 (기울기 dF/dt 기반) |
|  |  | **3240~3260** | **Dec L/M/H:** 감속 등급 (기울기 dF/dt 기반) |
|  |  | **3270~3290** | **Cyclic L/M/H:** 반복 빈도 (주기성 기반) |
|  |  | **3300** | **Coasting Down:** 전원 차단 후 관성 정지 |
|  | 미분류 (390) | **3910** | **Run Unknown:** 가동 중 미정의 패턴 |
| **UNKNOWN (90)** | 없음 (0) | **9000** | **L1 Unknown:** 시스템 판단 불가 상태 |
### 3. 상태 판단 지표 및 공식 (Features)
수식은 일반 텍스트로 표기하며, 윈도우 내 데이터를 기반으로 산출합니다.
 1. **RMS (Root Mean Square):** 전체 진동 에너지 실효치
 2. **Top-1 Peak Freq (F):** 스펙트럼에서 에너지가 가장 큰 주파수 (Hz)
 3. **dF/dt Slope (S):** 윈도우 내 주파수 변화량에 대한 선형 회귀 기울기
   * S = (W_size * sum(t * F) - sum(t) * sum(F)) / (W_size * sum(t * t) - (sum(t) * sum(t)))
 4. **R-Square (R2):** 주파수 추세의 선형성 (0~1)
   * 가감속의 명확성을 판별 (R2 > 0.7 이면 명확한 가감속)
 5. **Zero-Crossing Rate of Delta F (ZCR):** 주파수 변화량의 부호 반전 빈도
   * 반복(Cyclic) 패턴 판별의 핵심 지표
 6. **Spectral Flatness:** 스펙트럼의 평탄도 (0~1)
   * 1에 가까울수록 백색 잡음(OFF), 0에 가까울수록 순음(RUN)
### 4. 히스테리시스 및 디바운스 정교화 설계
상태 전이 시 Chattering(잦은 변경)을 방지하기 위한 이중 방어 전략입니다.
#### 4.1 공간적 히스테리시스 (Threshold Margin)
 * **에너지 경계:** Upper_Th = Base_Th * 1.2, Lower_Th = Base_Th * 0.9
 * **속도 등급 경계:** 구간 경계 주파수에서 +/- 5% 마진 밴드 적용
 * **가속 등급 경계:** High/Mid/Low 등급 간 전이 시 해당 등급 하한선의 10% 마진 적용
#### 4.2 시간적 디바운스 (Temporal Confirmation)
상태가 바뀌기 위해 해당 조건이 연속으로 만족되어야 하는 프레임 수입니다.
| 전이 경로 | 디바운스 (Frames) | 확정 시간 (초) |
|---|---|---|
| **OFF -> IDLE** | 5 프레임 | 약 0.8초 |
| **IDLE -> OFF** | 20 프레임 | 약 3.2초 (관성 고려) |
| **IDLE -> RUN** | 10 프레임 | 약 1.6초 |
| **STEADY -> TRANS** | 3 프레임 | 약 0.48초 (즉각 감지) |
| **TRANS -> STEADY** | 15 프레임 | 약 2.4초 (안정화 확인) |
### 5. 세부 상태별 판단 로직 (Sequential Filter)
**Step 1: Level 1 에너지 필터**
 * if RMS < Floor_Noise_Th AND Flatness > 0.8 -> **OFF (10)**
 * else if RMS < Run_Min_Th -> **IDLE (20)**
 * else -> **RUN (30)**
**Step 2: Level 2 & 3 주파수 추세 필터 (RUN 상태 시)**
 * if abs(S) < Stability_Th -> **STEADY (310)**
   * F 위치에 따라 LOW(3110) / MID(3120) / HIGH(3130) 결정
 * else (abs(S) >= Stability_Th) -> **TRANS (320)**
   * if ZCR > Cyclic_Th -> **CYCLIC (3270~90)** (ZCR 빈도에 따라 L/M/H)
   * else if S > 0 -> **ACC (3210~30)** (S 크기에 따라 L/M/H)
   * else if S < 0
     * if RMS_Slope < 0 AND Log_Linearity > 0.8 -> **COASTING (3300)**
     * else -> **DEC (3240~60)** (abs(S) 크기에 따라 L/M/H)
### 6. 독립 속성 플래그 (Attribute Flags - 중복 허용)
상태와 별개로 병렬 체크하여 비트 플래그로 저장합니다.
| 속성명 | 판별 조건 | 비고 |
|---|---|---|
| **is_loaded** | RMS > (State_Mean_RMS + 1.2 * State_StdDev) | 부하 가동 여부 |
| **is_anomaly** | Feature > (State_Mean + Confidence_k * State_StdDev) | **이상 징후 (즉시 트리거)** |
| **is_clipping** | Raw_Sample > Full_Scale * 0.98 | 센서 포화 경고 |
| **has_resonance** | Current_Freq가 사전에 정의된 Resonance_Band 포함 시 | 공진 구간 통과 |
| **is_unstable** | R-Square < 0.4 인 과도 상태 또는 S 분산 급증 시 | 제어 불안정 |
### 7. 구현 시 주의사항 (Developer Notes)
 1. **Candidate State 구조:** current_state와 별개로 candidate_state 변수를 운영하십시오. 디바운스 카운터가 충족되는 순간에만 current_state = candidate_state로 업데이트합니다.
 2. **Anomaly Hold Time:** is_anomaly 플래그가 한번 세워지면, 신호가 정상으로 돌아와도 최소 **5,000ms(5초)** 동안은 플래그를 유지하여 이벤트 파일의 무결성을 확보합니다.
 3. **Welford 알고리즘:** 각 상태별 Mean과 StdDev 프로파일링 시 메모리 절약을 위해 데이터를 쌓지 말고 프레임 단위로 실시간 업데이트하는 Welford 방식을 사용하십시오.

### 3. 상태 판단 지표 및 연산 공식
판단은 상위 레벨에서 하위 레벨로 내려가는 순차적 필터링 구조를 따릅니다.
#### [Step 1] 에너지 및 평탄도 (L1 판단)
 * **Total RMS:** 전체 주파수 대역의 에너지 실효치.
 * **Spectral Flatness:** 스펙트럼의 기하평균 / 산술평균.
   * 공식: Flatness = exp(sum(ln(bin_i)) / n) / (sum(bin_i) / n)
   * 해석: 1.0에 가까우면 백색 잡음, 0에 가까우면 순음(회전체).
#### [Step 2] 주파수 추세 분석 (L2/L3 가감속 판단)
 * **dF/dt Slope (기울기):** 윈도우 내 주파수 피크값들에 대한 선형 회귀 기울기.
   * 공식: Slope = (n * sum(t*F) - sum(t) * sum(F)) / (n * sum(t^2) - (sum(t))^2)
 * **R-Square (선형성):** 기울기 추정의 신뢰도.
   * 공식: R2 = 1 - (sum((F_obs - F_pred)^2) / sum((F_obs - F_mean)^2))
   * 해석: 0.7 이상이면 선형 가감속(Acc/Dec), 낮으면 불규칙(Random/Cyclic).
#### [Step 3] 주기성 분석 (Cyclic 판단)
 * **Autocorrelation (자기상관):** Delta F 신호의 주기성 확인.
   * 공식: ACF(k) = sum((X_i - mean) * (X_{i+k} - mean)) / Variance
   * 해석: 지연 시점(k)에서 피크 발생 시 해당 주기를 Cyclic 빈도로 확정.

### 5. 독립적 속성 (Attribute/Flag) 판단 로직
상태(State)와 별개로 중복 활성화될 수 있는 특이 현상들입니다.
 1. **Load (부하):** 동일 RPM 등급의 무부하 평균(mu) 대비 RMS가 20% 이상 상승 시 발생.
 2. **Resonance (공진):** 특정 주파수 대역 통과 시 에너지가 평소 mu + 6*sigma 이상 폭발적으로 증가 시 마킹.
 3. **Anomaly (이상):** 상태별 적응형 임계값 mu + k * sigma를 2프레임 연속 초과 시 즉시 발동.
 4. **Clipping (포화):** Raw 데이터 중 ADC 최대값의 98%에 도달한 샘플이 프레임 내 5% 초과 시 경고.
 5. **Noise (외란):** 주파수 피크는 없으나 에너지만 광대역으로 상승하는 경우.

 * **윈도우 관리:** 링버퍼(Circular Buffer)를 사용하여 최근 W_size 프레임의 피처 데이터를 유지.
 * **상태 확정 프로세스:** 1.  매 프레임 피처 추출.
   2.  L1 ➔ L2 ➔ L3 순서로 판단 로직 수행.
   3.  판단 결과가 이전 프레임과 다를 경우 Candidate 상태로 등록하고 디바운스 카운터 가동.
   4.  카운터 충족 시 Active_State 변경 및 Adaptive_Threshold 갱신.
 * **이벤트 기록:** is_anomaly == true 시 트리거 발동. 신호가 정상으로 복귀해도 최소 5초간 Hold_Time 유지.

## old history ***

T20 시스템의 설비 상태 인식(Machine State Awareness)을 고도화하기 위한 **세부 상태 정의와 판단 로직 설계**를 진행하겠습니다.
단순한 ON/OFF를 넘어, 설비의 물리적 동작 특성(RPM, 부하량, 전이 속도)을 피쳐 도메인에서 구분함으로써 **"정상적인 변화"와 "이상 진동"을 완벽히 분리**하는 것이 이번 설계의 핵심입니다.
### 1. 설비 상태 모드(States) 세분화 및 판단 방법
#### [A] OFF (Power Off)
 * **정의:** 설비에 전원이 완전히 차단된 상태.
 * **판단 방법:** * Total RMS < Sensor Floor Noise (사전에 학습된 센서 자체 노이즈 하한선).
   * Spectral Flatness ≈ 1.0 (특정 주파수 집중 없이 백색 잡음만 존재).
 * **비고:** 이 상태에서는 모든 트리거 감시를 중단하여 전력을 극단적으로 보존합니다.
#### [B] IDLE (Standby & Ready)
 * **추가 구분:**
   1. **IDLE_QUIET:** 제어 전원만 인가됨. 냉각 팬 등 미세 진동도 없는 상태.
   2. **IDLE_ACTIVE:** 메인 모터는 정지했으나 냉각 팬, 유압 펌프 등 보조 장치가 가동 중인 상태.
 * **판단 방법:**
   * Floor Noise < Total RMS < Run Threshold.
   * Top-1 Peak Frequency가 전원 주파수(60Hz) 또는 팬 회전수에 고정됨.
   * **IDLE_ACTIVE**는 특정 저주파 대역에 약한 에너지가 지속 관찰됨.
#### [C] RUN_STABLE (Steady State)
 * **정의:** 설정된 목표 속도로 정속 주행 중인 상태.
 * **판단 방법:** |ΔF (Top-1 Freq)| < 안정 한계치 (주파수 변동이 거의 없음).
 * **세부 등급 구분:**
   1. **Speed Grading:** Top-1 Peak의 주파수를 RPM으로 환산하여 등급화 (예: Low < 30Hz, Mid 30~50Hz, High > 50Hz).
   2. **Load vs. No-Load:** * **No-Load (무부하):** 정속 주행 중 Top-1 Amp 및 Total RMS가 기준 프로파일의 하한선에 위치.
     * **Loaded (부하):** 동일 RPM에서 무부하 대비 Total RMS가 유의미하게 상승하고, Harmonic Energy Ratio(고조파 에너지 비율)가 증가함.
#### [D] RUN_TRANSIENT (Transition State)
 * **정의:** 속도가 변하거나 부하가 급변하는 구간.
 * **판단 방법:** |ΔF (Top-1 Freq)| > 전이 한계치 또는 Spectral Flux 폭증.
 * **세부 구분:**
   1. **Acceleration (가속):** ΔF > 0이 일정 프레임 유지.
   2. **Deceleration (감속):** ΔF < 0이 일정 프레임 유지.
   3. **Cyclic/Pattern (반복):** 가/감속이 짧은 주기로 반복됨 (Inching 또는 간헐적 작업).
   4. **Transition Grade:** |ΔF|의 기울기(Slope)로 급가속/완만가속 구분.
   5. **Load During Transition:** Δ(Peak Amplitude)의 변화율을 통해 과부하 상태에서 가속 중인지 판단.
#### [E] 추가 검토 상태
 * **COASTING DOWN:** 전원은 차단되었으나 관성에 의해 회전하며 멈추는 상태 (감속과 유사하나 고유의 지수적 감쇠 패턴 존재).
 * **RESONANCE (공진):** 특정 RPM 등급에서만 진폭이 비정상적으로 커지는 구간 (고장으로 오판하기 쉬우므로 별도 프로파일링 필요).
### 2. 설비 상태 판단 로직트리 (Logic Tree)
설비의 상태를 결정하기 위한 순차적 의사결정 구조입니다. 이 로직은 T231_Dsp_Pipeline에서 매 프레임(1600Hz 기준 256샘플마다) 수행됩니다.
 1. **Level 1: 에너지 유무 검사**
   * Total\_RMS < Floor\_Noise ?
     * YES → **[OFF]**
     * NO → Go to Level 2
 2. **Level 2: 회전 성분 검사**
   * Spectral\_Flatness > 0.8 (매우 평탄) OR Top1\_Amp < Run\_Threshold ?
     * YES → **[IDLE]** * 보조 주파수 성분 존재? YES → **[IDLE_ACTIVE]** / NO → **[IDLE_QUIET]**
     * NO → Go to Level 3 (가동 중 확정)
 3. **Level 3: 속도 안정성 검사 (\Delta F)**
   * |\Delta F (Top1)| < \epsilon (안정) ?
     * YES → **[RUN_STABLE]**
       * RMS vs Baseline(\mu, \sigma) 비교 → **[Loaded]** or **[No-Load]**
       * Freq 값에 따른 등급 부여 → **[Low / Mid / High Speed]**
     * NO → **[RUN_TRANSIENT]**
       * \Delta F > 0 → **[Acceleration]** (기울기에 따라 Grade 부여)
       * \Delta F < 0 → **[Deceleration]**
       * 부하 지표 병행 확인
 4. **Level 4: 프로파일 이탈 검사 (Final Filter)**
   * 위 상태 중 어디에 해당하더라도, 해당 상태의 **적응형 임계값(\mu + k\sigma)**을 초과하는 피쳐가 발생했는가?
     * YES → **[ANOMALY]** 판정 및 이벤트 로그 생성
### 3. 향후 설계 방향 (Hysteresis 연계)
상기 상태 모드 구분이 완료되면, **트리거 히스테리시스**는 다음과 같이 더욱 강력해집니다.
 * **상태별 히스테리시스 차등화:**
   * **IDLE → RUN 전이 시:** 기동 초기 충격을 오알람으로 잡지 않도록 **디바운스 카운터**를 길게 설정.
   * **RUN_STABLE 상태 시:** 아주 미세한 이상도 포착해야 하므로 **히스테리시스 밴드**를 좁게 설정하여 감도 극대화.
   * **TRANSIENT 상태 시:** 가감속 중의 진동 폭증은 정상이므로, **임계값을 일시적으로 상향(Masking)**하거나 트리거를 유예.
이 로직트리를 바탕으로 T231_Dsp_Pipeline의 피쳐 추출부와 T230_Mfcc_Cor의 상태 전이 엔진을 업데이트할 준비가 되었습니다. 우선 **"Level 3까지의 상태 판별 알고리즘"** 구현을 위해 코드 작성을 시작할까요?







요청하신 사항을 엄격하게 반영하여, **설비 상태와 판단 방법, 그리고 히스테리시스/디바운싱 전략을 4개의 열(Column)로 명확히 분리한 마스터 테이블**을 재구성했습니다. 가속, 감속, 가/감속 반복 역시 '상태 소분류' 컬럼에 독립적으로 배치하였습니다.
### 1. 설비 상태 판단 로직 및 히스테리시스/디바운싱 마스터 테이블

# 추가 설계 메모

## 히스테리시스 공통 전략
- Enter Threshold ≠ Exit Threshold 구조 적용
- 상태 전이 조건:
  - Enter: Feature > Threshold_high
  - Exit: Feature < Threshold_low
- Threshold_high = 기준값 + delta
- Threshold_low = 기준값 - delta

## 디바운스 공통 전략
- 상태 전이는 반드시 "연속 프레임 조건" 만족 필요
- 기본 구조:
  - if condition == TRUE for N frames → 상태 변경
- 상태별 권장 프레임:
  - OFF/IDLE: 10~20
  - RUN 진입: 20~30
  - TRANSIENT: 5~10

## TRANSIENT 구간 보호 (핵심)
- masking_factor 적용:
  - effective_threshold = base_threshold * (1 + beta)
- 또는 anomaly detection 일시 hold

## 최종 구조 요약
1. 상태 판별 (Logic Tree)
2. 상태별 히스테리시스 적용
3. 디바운스 필터 통과
4. 이상 탐지 (상태 기반 adaptive threshold)

### 2. 설비 상태 관련 구조체 초안 설계 v2(C++)

```cpp
#pragma once

#include <stdint.h>

/**
 * @brief 설비 가동 상태 대분류 (Large Category)
 */
enum EM_T20_MachineState {
    EN_T20_STATE_OFF = 0,           // 전원 차단
    EN_T20_STATE_IDLE,              // 대기/준비
    EN_T20_STATE_RUN_STABLE,        // 정속 가동
    EN_T20_STATE_RUN_TRANSIENT,     // 과도 상태 (가감속)
    EN_T20_STATE_ANOMALY            // 이상 발생
};

/**
 * @brief 설비 상태 소분류 (Small Category)
 */
enum EM_T20_SubState {
    EN_T20_SUB_NONE = 0,
    // IDLE
    EN_T20_SUB_IDLE_QUIET,          // 정적 대기
    EN_T20_SUB_IDLE_ACTIVE,         // 보조기기 가동 중
    // RUN_STABLE
    EN_T20_SUB_STABLE_NOLOAD,       // 정속 무부하
    EN_T20_SUB_STABLE_LOADED,       // 정속 부하
    // RUN_TRANSIENT (세분화됨)
    EN_T20_SUB_TRANS_ACCEL,         // 가속 구간
    EN_T20_SUB_TRANS_DECEL,         // 감속 구간
    EN_T20_SUB_TRANS_CYCLIC         // 가/감속 반복 (Inching)
};

/**
 * @brief 설비 상태 인식 및 히스테리시스 관리 컨텍스트
 */
struct ST_T20_MachineStateCtx_t {
    // [1. 현재 상태]
    EM_T20_MachineState group;      // 대분류
    EM_T20_SubState     sub;        // 소분류

    // [2. 판단 지표]
    float rms_now;                  // 현재 실효치
    float freq_peak;                // 메인 주파수 피크 (Hz)
    float freq_delta;               // 이전 프레임 대비 주파수 변화량
    float flatness;                 // 스펙트럼 평탄도
    float peak_hold_max;            // 충격 탐지용 최대 절대값

    // [3. 디바운싱 카운터]
    uint16_t confirm_cnt;           // 상태 확정용 연속 카운터
    uint16_t transition_cnt;        // 상태 전이 대기 카운터
    uint16_t sign_flip_cnt;         // 주파수 변화 부호 반전 횟수 (반복 판별용)

    // [4. 히스테리시스 및 트리거 전략]
    bool     is_latched;            // 트리거 실행 중 여부
    uint32_t hold_timer_ms;         // 트리거 해제 유예 시간
    float    active_threshold;      // 현재 상태 가변 임계값 (mu + k*sigma)
    float    margin_ratio;          // 전이 방지 마진 (기본 0.15)

    ST_T20_MachineStateCtx_t() {
        group = EN_T20_STATE_OFF;
        sub = EN_T20_SUB_NONE;
        rms_now = 0.0f;
        freq_peak = 0.0f;
        freq_delta = 0.0f;
        flatness = 1.0f;
        peak_hold_max = 0.0f;
        confirm_cnt = 0;
        transition_cnt = 0;
        sign_flip_cnt = 0;
        is_latched = false;
        hold_timer_ms = 0;
        active_threshold = 0.0f;
        margin_ratio = 0.15f;
    }
};






### 2. 설비 상태 관련 구조체 초안 설계 v1(C++)
위 테이블의 세분화된 상태와 디바운싱/히스테리시스 로직을 온전히 담아낼 수 있는 C++ 구조체(ST_T20_MachineStateCtx_t) 초안입니다.
```cpp
#pragma once

#include <stdint.h>
#include "T210_Def_231.h"

// =========================================================================
// 1. 상태 분류 ENUM
// =========================================================================

/**
 * @brief 설비 가동 상태 대분류 (State Group)
 */
enum EM_T20_MachineState {
    EN_T20_STATE_OFF = 0,
    EN_T20_STATE_IDLE,
    EN_T20_STATE_RUN_STABLE,
    EN_T20_STATE_RUN_TRANSIENT,
    EN_T20_STATE_ANOMALY
};

/**
 * @brief 설비 상태 소분류 (Specific State)
 */
enum EM_T20_SubState {
    EN_T20_SUB_NONE = 0,
    // IDLE 세부
    EN_T20_SUB_IDLE_QUIET,
    EN_T20_SUB_IDLE_ACTIVE,
    // RUN_STABLE 세부
    EN_T20_SUB_STABLE_NOLOAD,
    EN_T20_SUB_STABLE_LOADED,
    // RUN_TRANSIENT 세부
    EN_T20_SUB_TRANS_ACCELERATION, // 가속
    EN_T20_SUB_TRANS_DECELERATION, // 감속
    EN_T20_SUB_TRANS_CYCLIC        // 가/감속 반복
};

// =========================================================================
// 2. 설비 상태 및 히스테리시스 관리 컨텍스트 구조체
// =========================================================================

struct ST_T20_MachineStateCtx_t {
    // [1. 현재 상태 정보]
    EM_T20_MachineState current_state;
    EM_T20_SubState     sub_state;
    
    // [2. 피쳐 기반 상태 판단 지표 (Features)]
    float current_rms;
    float dominant_freq;       // Top-1 Peak Frequency
    float delta_freq;          // 이전 프레임 대비 주파수 변화량 (가/감속 판별용)
    float spectral_flatness;   // 노이즈/순음 판별
    float peak_hold_max;       // 충격(Impact) 탐지용 Raw Max 값

    // [3. 디바운싱 카운터 (Debouncing)]
    // 노이즈로 인한 일시적인 값 튐 현상을 방지하기 위해 일정 프레임 이상 조건이 지속되어야 함
    uint16_t stable_frames_cnt;     // 현재 상태 조건이 연속으로 유지된 프레임 수
    uint16_t transition_frames_cnt; // 다른 상태로 전이하기 위한 조건이 연속 감지된 프레임 수
    uint16_t cyclic_pattern_cnt;    // 가감속 교차(부호 반전) 횟수 누적 카운터

    // [4. 히스테리시스 및 트리거 유지 (Hysteresis & Latch)]
    bool     is_latched;            // 트리거(기록) 발동 여부 
    uint32_t latch_hold_timer_ms;   // 트리거 해제 전 유지 시간 (채터링 방지)
    float    adaptive_threshold;    // 현재 상태(State)에 맞춰 스위칭된 가변 임계값
    float    hysteresis_margin;     // 상태 이탈을 막기 위한 여유분 (예: 0.15)

    // 생성자 (초기화)
    ST_T20_MachineStateCtx_t() {
        current_state = EN_T20_STATE_OFF;
        sub_state = EN_T20_SUB_NONE;

        current_rms = 0.0f;
        dominant_freq = 0.0f;
        delta_freq = 0.0f;
        spectral_flatness = 1.0f;
        peak_hold_max = 0.0f;

        stable_frames_cnt = 0;
        transition_frames_cnt = 0;
        cyclic_pattern_cnt = 0;

        is_latched = false;
        latch_hold_timer_ms = 0;
        adaptive_threshold = 0.0f;
        hysteresis_margin = 0.15f; // 15% 마진 기본값
    }
};

```
