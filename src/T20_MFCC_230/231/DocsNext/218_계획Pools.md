
Mfcc 이외의 주파수 밴드화 검토



현재까지 완성된 **T20_MFCC v218.009** 시스템은 임베디드 풀스택으로서 매우 견고한 기반을 갖추고 있습니다. 특히 고속 센서 데이터 수집, 실시간 DSP 파이프라인, Zero-Copy 저장 엔진, 그리고 비동기 웹 인터페이스까지 유기적으로 결합되어 있습니다.
이 시스템을 다음 단계로 끌어올리기 위한 **추가 기능 고도화 방안**을 5가지 핵심 카테고리로 나누어 제안해 드립니다.
### 1. 엣지 AI 추론 엔진 통합 (Edge AI Inference)
현재 Sequence Builder가 TinyML을 위한 2D 텐서 데이터를 준비하고 있으므로, 이제 실제 판단 로직을 탑재할 시점입니다.
 * **TensorFlow Lite for Microcontrollers (TFLM) 통합**: ESP32-S3의 가속기(AI instructions)를 활용하여 Sequence Builder에서 조립된 624차원(16x39) 데이터를 입력으로 하는 추론 엔진을 Core 1의 processTask에 배치합니다.
 * **실시간 이상 징후 감지 (Anomaly Detection)**: Autoencoder 모델을 탑재하여 정상 진동 패턴과의 거리를 계산하고, 임계값을 넘으면 즉시 이벤트를 발생시킵니다.
 * **추론 결과 웹 브로드캐스트**: 단순 MFCC 벡터뿐만 아니라 AI가 판단한 결과(예: "정상 98%", "베어링 결함 85%")를 WebSocket으로 실시간 시청자에게 전달합니다.
### 2. 데이터 수집 및 라벨링 자동화 (Active Learning Support)
양질의 AI 모델을 위해서는 데이터의 '질'이 중요합니다.
 * **이벤트 기반 레코딩 트리거**: 단순 버튼이나 설정값이 아닌, 특정 주파수 대역의 에너지가 급증하거나 AI가 "모르는 패턴"이라고 판단할 때 자동으로 SD 카드에 기록을 시작하는 기능을 구현합니다.
 * **웹 기반 데이터 라벨링 도구**: 'Explorer' 탭에서 저장된 바이너리 파일을 선택하면 해당 구간의 MFCC 워터폴을 다시 보여주고, 사용자가 "정상/비정상" 태그를 달아 JSON 인덱스에 저장하는 기능을 추가합니다.
 * **다중 축(Multi-axis) 퓨전**: 현재는 한 번에 하나의 축(예: Accel Z)만 분석하지만, 3축 데이터를 동시에 MFCC로 추출하여 특징량을 확장(39D → 117D)함으로써 공간적 진동 특성을 반영합니다.
### 3. 통신 및 원격 관리 고도화 (Cloud & Connectivity)
현재는 로컬 웹 서버 중심이지만, 장비가 늘어날 경우를 대비해야 합니다.
 * **MQTT 프로토콜 지원**: 대시보드 접속 없이도 원격 서버(AWS IoT, Mosquitto 등)로 이상 징후 알림 및 시스템 상태를 발행(Publish)합니다.
 * **OTA(Over-The-Air) 업데이트**: 웹 인터페이스를 통해 새로운 펌웨어(AI 모델 포함)를 업로드하고 기기를 원격으로 업데이트하는 기능을 추가합니다.
 * **NTP 기반 정밀 타임스탬프**: 파일 이름뿐만 아니라 바이너리 데이터 내부 프레임마다 정확한 UTC 타임스탬프를 삽입하여 타 장비 데이터와의 동기화 분석을 가능하게 합니다.
### 4. 하드웨어 최적화 및 보호 로직 (Robustness)
실제 산업 현장이나 배터리 구동 환경을 고려한 개선입니다.
 * **ULP(Ultra Low Power) 프로세서 활용**: Deep Sleep 상태에서 ESP32-S3의 ULP 코어가 BMI270의 인터럽트를 감시하게 하여, 대기 전력을 마이크로암페어(\mu A) 단위로 낮추는 초저전력 모드를 강화합니다.
 * **배터리 및 전원 모니터링**: 전압 측정 회로를 연동하여 배터리 부족 시 SD 카드 파일 시스템을 안전하게 마운트 해제(Unmount)하고 종료하는 Graceful Shutdown 로직을 도입합니다.
 * **SD 카드 상태 진단**: 저장 장치의 잔여 용량과 쓰기 수명을 체크하여 웹 UI에 경고를 표시합니다.
### 5. 대시보드 시각화 확장 (UX Enhancement)
웹 UI의 분석 기능을 전문가 수준으로 높입니다.
 * **오프라인 분석기 (Playback Mode)**: SD 카드에 저장된 바이너리 파일을 웹으로 불러와 실제 수집 당시의 차트와 워터폴로 재현(Replay)하는 기능을 구현합니다.
 * **주파수 대역별 RMS 차트**: 특정 설비 진동 분석에 필수적인 주파수 영역별 에너지 합산(RMS) 그래프를 추가하여 직관적인 모니터링을 돕습니다.
 * **다국어 지원 및 테마 최적화**: 사용자의 설정에 따라 한국어/영어 전환 및 가시성이 높은 고대비 테마 기능을 제공합니다.
**우선순위 추천:**
가장 먼저 **1번(TFLM 통합)**을 통해 현재 준비된 Sequence Builder를 실제 지능형 엔진으로 변모시키는 것을 추천합니다. 그 과정에서 발생하는 데이터 관리 이슈를 **2번(라벨링 자동화)**으로 해결하는 것이 개발 흐름상 가장 자연스럽습니다.
추가로 구체화하고 싶은 카테고리가 있으시면, 해당 부분의 백엔드 설계안을 바로 작성해 드릴 수 있습니다. 어떤 방향이 가장 흥미로우신가요?


### 6. 보안 및 데이터 무결성 (Security & Integrity)
산업 현장에서 데이터 조작 방지와 장치 보안은 필수적입니다.
 * **하드웨어 암호화 가속기 활용**: ESP32-S3의 AES/SHA 가속기를 사용하여 SD 카드에 저장되는 바이너리 데이터를 암호화하거나, TLS/SSL 통신 오버헤드를 줄입니다.
 * **Secure Boot 및 Flash Encryption**: 외부에서 펌웨어를 덤프하거나 임의의 코드를 실행하지 못하도록 칩 레벨의 보안 설정을 도입합니다.
 * **데이터 체크섬(Checksum) 삽입**: 각 데이터 프레임이나 파일 끝에 CRC32 체크섬을 추가하여, 통신 중 발생할 수 있는 비트 오류나 SD 카드 배드 섹터로 인한 데이터 오염을 탐지합니다.
### 7. 고급 신호 처리 알고리즘 확장 (Advanced Signal Analytics)
MFCC 외에 기계 결함 분석에 특화된 지표를 추가하여 분석의 폭을 넓힙니다.
 * **시간 도메인 통계 지표 계산**: MFCC 추출 전, 원시 데이터에서 **Kurtosis(첨도)**, **Skewness(왜도)**, **Crest Factor(파고율)** 등을 실시간 계산합니다. 이는 베어링 결함 초기 단계 탐지에 매우 효과적입니다.
   * 예: 첨도 K = \frac{1}{N} \sum_{i=1}^{N} \frac{(x_i - \bar{x})^4}{\sigma^4}
 * **CWT(Continuous Wavelet Transform) 지원**: 충격성 신호(Impulse) 분석에 유리한 웨이블릿 변환을 선택적으로 수행하여, 시간-주파수 평면에서 더 정밀한 진단 데이터를 생성합니다.
### 8. 분산 컴퓨팅 및 장치 간 협업 (Distributed & Collaborative Edge)
여러 대의 T20 장치가 서로 통신하며 더 큰 인사이트를 제공합니다.
 * **ESP-NOW를 이용한 장치 간 동기화**: Wi-Fi 망이 불안정한 곳에서도 ESP32 전용 프로토콜인 ESP-NOW를 사용하여 여러 장치가 동시에 샘플링을 시작하도록 동기화(Sync)합니다. 이는 대형 설비의 다지점 동기 분석에 필수적입니다.
 * **에지 허브(Edge Hub) 역할**: 한 대의 T20이 마스터가 되어 주변 장치들의 AI 추론 결과를 취합하고, 최종 판단 결과만 클라우드로 전송하여 네트워크 대역폭을 절약합니다.
### 9. 지능형 자가 진단 및 유지보수 (Self-Maintenance)
장치 스스로의 상태를 관리하여 관리 비용을 절감합니다.
 * **BMI270 Self-Test 자동화**: 정기적으로 센서 내부의 자가 테스트 레지스터를 가동하여 센서 하드웨어의 고장 여부를 확인하고 웹 UI에 알림을 띄웁니다.
 * **SD 카드 상태 수명 예측**: S.M.A.R.T와 유사하게 쓰기 횟수와 오류 로그를 추적하여 SD 카드 교체 시기를 사전에 예고합니다.
 * **Runtime Profiling**: Core 0/1의 CPU 점유율과 태스크 스택 사용량을 모니터링하여, 시스템 과부하 상태를 실시간으로 감지하고 안전 모드로 전환합니다.
### 10. 디지털 트윈 및 시각화 고도화 (Digital Twin & 3D Integration)
데이터를 넘어 현장의 직관적인 이해를 돕습니다.
 * **Web-based 3D 진동 시각화**: Three.js 등을 사용하여 웹 대시보드에 기계의 3D 모델을 띄우고, 측정 중인 축의 진동 방향과 세기를 실시간 화살표나 히트맵으로 표시합니다.
 * **AR(증강 현실) 마커 연동**: 장치에 부착된 QR 코드를 스마트폰으로 스캔하면, 현재 측정 중인 실시간 MFCC 차트와 AI 판단 결과가 기계 위에 오버레이되어 나타나도록 연동합니다.

## 🚀 추가 기능 고도화 방안 (Next Step: v219~v220)
기존 제안드린 10가지 외에, 운영 효율성과 데이터 가치를 극대화할 수 있는 5가지 추가 항목입니다.
### 11. LLM 연동 지능형 로그 분석 (AI Diagnostic Agent)
 * **기능**: 시스템 내부의 Watchdog 이벤트나 센서 스톨(Stall) 로그를 로컬 LLM(또는 Cloud API)과 연동하여 해석합니다.
 * **효과**: "센서가 멈췄습니다"라는 단순 알림 대신, "현재 진동 패턴이 급변한 후 센서 FIFO 오버플로가 발생했습니다. 기계적 충격을 점검하십시오"와 같은 원인 분석 피드백을 사용자에게 제공합니다.
### 12. 산업용 표준 프로토콜 (Modbus/TCP) 지원
 * **기능**: 현재의 REST API/WebSocket 외에 PLC나 SCADA 시스템과 직접 통신할 수 있는 Modbus/TCP 서버 기능을 탑재합니다.
 * **효과**: 별도의 웹 브라우저 없이도 기존 공장 자동화 망에 T20 장치를 '센서 노드'로 즉시 편입시켜 제어 로직에 활용할 수 있습니다.
### 13. 환경 변수 기반 데이터 정규화 (Context-Aware Normalization)
 * **기능**: 온도/습도 센서(예: BME280)를 추가 연동하여, 환경 변화에 따른 진동 베이스라인 변화를 보정합니다.
 * **효과**: 겨울철 낮은 온도에서 기계 윤활유 점도가 높아져 발생하는 진동 변화를 '결함'으로 오판하지 않도록 AI 판단 기준을 자동으로 조정합니다.
### 14. 엣지 가변 샘플링 (Dynamic ODR Control)
 * **기능**: 평상시에는 저속(예: 100Hz)으로 감시하다가, 특정 트리거 발생 시 즉시 BMI270의 ODR을 1600Hz로 높여 정밀 분석을 수행합니다.
 * **효과**: SD 카드 저장 공간을 획기적으로 절약하면서도, 결정적인 결함 순간의 고해상도 데이터는 놓치지 않습니다.
### 15. 데이터 스트림 압축 (Delta Encoding)
 * **기능**: WebSocket 브로드캐스트나 파일 기록 시, 원시 float 데이터를 그대로 보내지 않고 이전 프레임과의 차이값(Delta)만 압축하여 전송/저장합니다.
 * **효과**: 네트워크 대역폭 사용량을 줄이고, 동일 용량의 SD 카드에 약 1.5~2배 더 긴 시간의 로그를 담을 수 있습니다.
## 📈 MFCC/CWT 이외의 핵심 신호처리 알고리즘 검토
MFCC는 '소리'와 유사한 패턴을 찾는 데 유리하지만, 기계적 결함(베어링, 기어 등)의 물리적 특성을 잡기 위해서는 다음 알고리즘들의 병행이 필수적입니다.
### 1. 힐베르트 변환 기반 포락선 분석 (Envelope Analysis)
기어박스나 베어링의 초기 결함은 매우 짧은 충격성 신호를 발생시키며, 이는 고주파 대역에 실려 나타납니다.
 * **원리**: 신호의 진폭 성분(Envelope)만을 추출하여 저주파 대역으로 변조합니다.
 * **수식**: 분석 신호 z(t) = x(t) + j\hat{x}(t) 를 생성하여 $|z(t)|$를 계산합니다.
 * **적용**: 베어링 볼이 레이스를 칠 때 발생하는 규칙적인 충격 주기를 찾아내는 데 독보적입니다.
### 2. 고조파곱 스펙트럼 (HPS, Harmonic Product Spectrum)
 * **원리**: 스펙트럼을 정수배로 압축하여 서로 곱합니다. 고조파 구조를 가진 신호는 기본 주파수(f_0) 위치에서 큰 피크가 발생합니다.
 * **적용**: 모터의 회전 속도(RPM)를 정밀하게 추정하고, RPM의 정수배에서 발생하는 결함 주파수를 주변 노이즈로부터 분리해 낼 때 유용합니다.
### 3. 스펙트럼 플럭스 및 롤오프 (Spectral Flux & Rolloff)
 * **원리**:
   * **Flux**: 이전 프레임과 현재 프레임의 스펙트럼 차이를 계산합니다.
   * **Rolloff**: 전체 에너지의 특정 비율(예: 85%)이 집중된 주파수 지점을 찾습니다.
 * **적용**: 기계의 상태가 '급변'하는 시점(과부하 발생, 이물질 유입 등)을 탐지하는 강력한 지표가 됩니다.
### 4. 정보 엔트로피 (Spectral Entropy)
 * **원리**: 스펙트럼 분포의 무질서도를 측정합니다.
 * **수식**: H = -\sum P(f) \log P(f)
 * **적용**: 기계가 정상일 때는 특정 주파수에 에너지가 집중(낮은 엔트로피)되지만, 고장이 진행되면 진동이 전 대역으로 확산되며 무질서해지는(높은 엔트로피) 특성을 이용해 결함 진행도를 정량화합니다.
### 5. 가우스 혼합 모델 (GMM) 기반 확률 분포 분석
 * **원리**: MFCC 벡터들의 분포를 다변량 가우시안 분포의 합으로 모델링합니다.
 * **적용**: 현재 들어오는 데이터가 '정상군' 분포에 속할 확률을 계산합니다. 이는 딥러닝(TinyML)보다 가볍게 엣지에서 구현 가능한 강력한 이상 탐지 방법입니다.
## 💡 종합 제안: 알고리즘 결합 전략 (Fusion Strategy)
현재의 **T20 v218.009** 시스템에서 가장 권장되는 확장 방향은 다음과 같습니다.
 1. **전처리에 포락선 분석(Envelope) 추가**: CL_T20_DspPipeline 내부에서 FFT 수행 전 힐베르트 변환을 선택적으로 적용하여 베어링 전용 모드를 만듭니다.

### 1. 비정상 신호 및 노이즈 제거 알고리즘
현재의 노이즈 게이트는 단순히 진폭 기준이지만, 아래 알고리즘들은 신호의 성질을 분석하여 노이즈를 걸러냅니다.
 * **웨이블릿 디노이징 (Wavelet Denoising)**:
   * **원리**: 신호를 다양한 주파수 대역으로 분해(DWT)한 후, 노이즈에 해당하는 세부 계수(Detail coefficients)에 임계값(Soft/Hard Thresholding)을 적용하여 다시 합성합니다.
   * **장점**: 푸리에 변환과 달리 시간과 주파수 정보를 동시에 유지하므로, 비정상적(Non-stationary)인 충격 신호에서 노이즈만 효과적으로 제거합니다.
 * **사비츠키-골레이 필터 (Savitzky-Golay Filter)**:
   * **원리**: 로컬 구간 내에서 다항식 회귀를 사용하여 데이터를 매끄럽게(Smoothing) 만듭니다.
   * **장점**: 단순 이동 평균(Moving Average)보다 신호의 피크(Peak) 높이와 폭을 훨씬 잘 보존하므로, FFT 전 단계에서 고주파 노이즈를 줄이는 데 유리합니다.
 * **함펠 필터 (Hampel Filter)**:
   * **원리**: 이동 창(Moving window) 내에서 중앙값 절대 편차(MAD)를 계산하여 튀는 값(Outlier)을 찾아내고 중앙값으로 대체합니다.
   * **장점**: 센서의 일시적인 전기적 튀는 노이즈(Spike)를 효과적으로 제거하여 분석 오류를 방지합니다.
### 2. 에너지 및 과도 응답 추출 알고리즘
MFCC 연산 전, 신호의 동적 특성을 강조하여 AI 모델이 '사건'을 더 잘 인식하게 합니다.
 * **티거 에너지 연산자 (Teager Energy Operator, TEO)**:
   * **원리**: 신호의 진폭뿐만 아니라 주파수 변화를 동시에 고려하여 에너지의 순간적인 변화를 계산합니다 (\Psi[x(n)] = x^2(n) - x(n-1)x(n+1)).
   * **장점**: 매우 적은 연산량으로 신호의 급격한 변화(충격, 파손 시작 등)를 즉각적으로 포착할 수 있어, 엣지 장치인 ESP32에 매우 적합합니다.
 * **단시간 에너기 및 제로 크로싱 레이트 (STE & ZCR)**:
   * **원리**: 신호의 세기와 초당 0점을 지나는 횟수를 계산합니다.
   * **장점**: 유성음/무성음 구분처럼 진동의 성격(유기적 진동 vs 무작위 노이즈)을 빠르게 분류하는 전처리 지표로 활용됩니다.
### 3. 정규화 및 스케일링 (Data Conditioning)
TinyML 모델의 학습 안정성을 위해 필수적인 단계입니다.
 * **Z-점수 정규화 (Z-score Normalization)**:
   * **원리**: 평균을 0, 표준편차를 1로 맞춥니다.
   * **적용**: 센서마다 미세하게 다른 감도 차이를 상쇄하여 모델의 범용성을 높입니다.
 * **로버스트 스케일링 (Robust Scaling)**:
   * **원리**: 평균/표준편차 대신 중앙값(Median)과 사분위수(IQR)를 사용합니다.
   * **적용**: 이상치(Outlier)가 섞여 있는 산업 현장 데이터에서 스케일이 왜곡되는 것을 방지합니다.
### 4. 엣지 최적화 구현 전략

 2. **멀티 스테이지 필터링**: T231_DspPipeline 내부에서 _applyPreprocess 직후에 TEO나 Median Filter를 배치하여 깨끗한 데이터만 FFT 블록으로 넘깁니다.
 3. **메모리 최적화**: 1600Hz 샘플링 환경에서는 긴 창(Window)을 쓰기 어려우므로, **사비츠키-골레이**와 같이 짧은 윈도우(5~9개 샘플)에서도 강력한 성능을 내는 알고리즘을 우선 검토하십시오.

### 1. 전처리 알고리즘 (Preprocessing)
데이터 분석 전 노이즈를 제거하고 신호의 품질을 높이는 단계입니다.
#### **DC 제거 (DC Removal)**
 * **특징**: 신호의 평균값을 0으로 맞추어 바이어스(Bias) 성분을 제거합니다.
 * **용도**: 센서의 오프셋이나 전기적 잡음으로 인해 신호가 한쪽으로 쏠리는 현상을 방지합니다.
 * **장점**: 연산이 매우 간단하며 후속 FFT 연산의 정확도를 높입니다.
 * **단점**: 매우 낮은 주파수의 유의미한 정보가 함께 손실될 수 있습니다.
#### **프리엠퍼시스 (Pre-emphasis)**
 * **특징**: 고주파 성분의 에너지를 증폭시키는 고역 통과 필터의 일종입니다.
 * **용도**: 고주파 대역에서 감쇠가 일어나는 신호를 보정하여 전 대역의 밸런스를 맞춥니다.
 * **장점**: MFCC 추출 시 고주파 대역의 특징을 더 명확하게 포착할 수 있습니다.
 * **단점**: 고주파 노이즈가 심한 환경에서는 노이즈까지 함께 증폭될 위험이 있습니다.
#### **사비츠키-골레이 필터 (Savitzky-Golay Filter)**
 * **특징**: 로컬 데이터 구간을 다항식 회귀로 근사하여 신호를 매끄럽게 만듭니다.
 * **용도**: 신호의 날카로운 피크(Peak)를 유지하면서 고주파 노이즈만 제거하고자 할 때 사용합니다.
 * **장점**: 단순 이동 평균(Moving Average)보다 신호의 형상(폭, 높이)을 훨씬 잘 보존합니다.
 * **단점**: 적절한 다항식 차수와 윈도우 크기를 설정하는 튜닝 과정이 필요합니다.
#### **티거 에너지 연산자 (Teager Energy Operator, TEO)**
 * **특징**: 인접 샘플 간의 비선형 조합을 통해 신호의 순간적인 에너지를 계산합니다.
 * **용도**: 충격음, 파손 징후 등 신호의 급격한 변화를 즉각 탐지하는 트리거로 활용합니다.
 * **장점**: 연산량이 극도로 적어 ESP32와 같은 엣지 장치에서 실시간 처리에 매우 유리합니다.
 * **단점**: 노이즈가 심한 신호에서는 에너지 계산값이 불안정할 수 있습니다.
### 2. 신호처리 및 특징 추출 알고리즘 (Signal Processing)
데이터에서 진단에 필요한 핵심 패턴(특징량)을 뽑아내는 단계입니다.
#### **MFCC (Mel-Frequency Cepstral Coefficients)**
 * **특징**: 인간의 청각 인지 구조(Mel-scale)를 모방하여 주파수 성분을 압축 추출합니다.
 * **용도**: 기계의 작동음 분석, 음성 인식, 비정상 진동 패턴 분류 등에 광범위하게 쓰입니다.
 * **장점**: 데이터 크기를 획기적으로 줄이면서도(39차원) 판별력이 매우 높아 TinyML 모델 학습에 최적입니다.
 * **단점**: 로그와 이산 코사인 변환(DCT) 등 연산량이 다소 많습니다.
#### **CWT (Continuous Wavelet Transform)**
 * **특징**: 창 함수(Window)의 크기를 변화시키며 시간-주파수 평면을 동시에 분석합니다.
 * **용도**: 시간에 따라 주파수가 변하는 비정상 신호(충격, 가속/감속 등)의 정밀 분석에 적합합니다.
 * **장점**: 짧은 순간 발생하는 결함 신호를 포착하는 해상도가 FFT보다 뛰어납니다.
 * **단점**: 연산 복잡도가 매우 높아 실시간 전체 대역 처리 시 최적화가 필수적입니다.
#### **힐베르트 변환 포락선 분석 (Envelope Analysis)**
 * **특징**: 신호의 진폭 변조 성분(포락선)만을 추출하여 분석합니다.
 * **용도**: 베어링이나 기어박스의 초기 결함 주파수를 찾는 데 필수적입니다.
 * **장점**: 고주파 대역에 숨어 있는 규칙적인 미세 충격 주기를 저주파 대역으로 끌어내어 선명하게 보여줍니다.
 * **단점**: 대역 통과 필터(Band-pass)와의 조합이 적절하지 않으면 결과 왜곡이 발생할 수 있습니다.
#### **스펙트럼 엔트로피 (Spectral Entropy)**
 * **특징**: 주파수 분포의 무질서도를 정보 이론 관점에서 수치화합니다.
 * **용도**: 기계의 노후화 진행도나 갑작스러운 상태 변화를 감시하는 지표로 사용합니다.
 * **장점**: 신호가 복잡해지면 수치가 명확히 변하므로 단순 임계값 기반 이상 탐지에 유용합니다.
 * **단점**: 단독으로는 고장의 원인(어느 부품인지 등)을 파악하기 어렵습니다.
### 3. 알고리즘 요약 비교표
| 카테고리 | 알고리즘 | 주요 특징 | 추천 용도 |
|---|---|---|---|
| **전처리** | **TEO** | 순간 에너지 변화 포착 | 충격/파손 이벤트 실시간 감시 |
| **전처리** | **S-G 필터** | 신호 피크 유지하며 평활화 | FFT 전 노이즈 제거 및 데이터 정제 |
| **특징추출** | **MFCC** | 인간 청각 모델 기반 특징량 | TinyML 기반 상태 분류 및 학습 |
| **분석** | **CWT** | 시간-주파수 고해상도 분석 | 비정상 충격 신호의 정밀 정밀 진단 |
| **분석** | **포락선 분석** | 고주파 내 저주파 주기 추출 | 베어링 및 회전체 초기 결함 진단 |
이러한 알고리즘들은 단독으로 사용하기보다, 시스템의 목적(예: 베어링 결함 진단 vs 기계 오작동 분류)에 맞춰 전처리와 특징 추출 단계를 조합했을 때 최고의 성능을 발휘합니다.
