v213 설계의 모든 컴파일 및 링킹 오류가 해결되어 **[빌드 가능한 베이스라인]**이 확보되었습니다. 이제 시스템의 완성도를 높이기 위해 현재 축소 구현된 기능을 점검하고, 1600Hz 고속 샘플링 환경에서의 성능 최적화 보완 계획을 제안합니다.
1. 미구현 및 축소 구현 기능 점검 (Gap Analysis)

2. 성능 최적화 검토 및 보완 계획
1600Hz(0.625ms 주기)는 ESP32-S3에서도 상당히 타이트한 일정입니다. 지터(Jitter)를 줄이고 처리 효율을 극대화하기 위한 계획입니다.
### 2.1. 센서 데이터 취득 최적화 (Latency)
 * 문제점: 현재 ISR에서 Task를 깨우는 과정에서 OS 스케줄링 오버헤드 발생 가능.
 * 보완:
   * Direct Task Notification 강화: vTaskNotifyGiveFromISR 사용 (현재 적용됨).
   * SPI DMA 전송: 센서 데이터를 읽어올 때 CPU를 사용하지 않고 DMA를 통해 메모리로 직접 전송(Bus 아비트레이션 최적화).
### 2.2. DSP 연산 병렬화 (Throughput)
 * 문제점: MFCC(FFT, Log-Mel, DCT) 연산이 ProcessTask 하나에서 순차적으로 실행됨.
 * 보완:
   * Dual Core 분산: Core 0은 센서/네트워크를 전담하고, Core 1은 DSP 연산 전용(Computation Intensive)으로 격리.
   * ESP-DSP Library 활용: 현재 수동으로 작성된 DCT-II/FFT 로직을 S3의 **SIMD 명령(Xtensa PIE)**을 사용하는 dsps_fft_f32_ae32 등으로 교체하여 연산 속도 3~5배 향상.
### 2.3. 저장소 쓰기 지연 방지 (Zero-Copy)
 * 문제점: SD카드 쓰기 중 발생하는 지연(최대 수십 ms)이 센서 수집에 영향을 줄 수 있음.
 * 보완:
   * Triple Buffering: 현재 2개인 DMA 슬롯을 3개 이상으로 늘려 SD카드 쓰기가 지연되어도 데이터 수집 버퍼가 마르지 않도록 마진 확보.
   * Pre-allocation: 파일 생성 시 미리 디스크 공간을 할당하여 파일 크기 증가에 따른 FAT 업데이트 오버헤드 최소화.
3. 향후 3단계 보완 로드맵
수립된 계획에 따라 다음 순서로 작업을 진행할 것을 권장합니다.
[Step 1] 하드웨어 가속기(SIMD) 통합
 * 가장 먼저 DSP 연산 부하를 줄여야 합니다. ESP-DSP 라이브러리를 통해 FFT 및 벡터 연산을 최적화하여 CPU 점유율을 20% 이하로 낮춥니다.
[Step 2] 센서 FIFO 엔진 고도화
 * 인터럽트 횟수를 획기적으로 줄이기 위해 FIFO Burst Read 로직을 완성합니다. 이는 전력 소모 감소와 시스템 안정성 향상에 직결됩니다.
[Step 3] 워치독 및 자가 진단 기능
 * 데이터 흐름 정체(Stall) 발생 시 센서를 자동으로 재초기화하는 **"Data Flow Watchdog"**을 추가하여 24시간 무중단 동작을 보장합니다.
