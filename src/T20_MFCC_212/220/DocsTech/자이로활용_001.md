설비 및 기계의 진동/소음 분석에서 **6축 IMU(Inertial Measurement Unit, 관성 측정 장치)** 센서를 사용할 때, 가속도계(3축) 외에 **자이로스코프(3축) 데이터가 왜 필요한지**에 대해 설명해 드리겠습니다.
전통적인 진동 분석은 주로 '선형 가속도'에 집중했지만, 현대의 정밀 진동 진단(PdM, 예지 보전)에서는 자이로 데이터를 통한 '각속도' 분석이 필수적인 요소로 자리 잡고 있습니다.
## 1. 선형 운동 vs 회전 운동의 보완
가속도계는 물체의 직선 운동을 측정하지만, 실제 설비의 결함은 **비틀림(Torsional)**이나 **회전축의 흔들림**으로 나타나는 경우가 많습니다.
 * **가속도계 (Accelerometer):** 상하, 좌우, 전후의 직선 진동을 측정합니다. (예: 베어링 마모로 인한 충격)
 * **자이로스코프 (Gyroscope):** 특정 축을 중심으로 회전하려는 속도(각속도)를 측정합니다. (예: 샤프트의 비틀림 진동)
## 2. 자이로 데이터가 꼭 필요한 이유
### ① 비틀림 진동(Torsional Vibration) 감지
모터나 엔진의 샤프트(축)는 회전하면서 미세하게 뒤틀리거나 회전 속도가 불규칙해질 수 있습니다. 이는 일반 가속도계로는 잡아내기 매우 어렵습니다. 자이로 센서는 **회전 속도의 미세한 변화(Fluctuation)**를 직접적으로 감지하여 기어박스의 결함이나 축의 불균형을 더 정확히 찾아냅니다.
### ② 센서 융합(Sensor Fusion)을 통한 데이터 정밀도 향상
가속도계는 중력의 영향을 크게 받으며, 고주파 진동에는 강하지만 저주파에서 위치 데이터가 흐르는(Drift) 경향이 있습니다. 반면 자이로는 짧은 시간의 회전 변화에 매우 민감합니다.
 * **보정 효과:** 자이로 데이터를 사용해 센서가 부착된 각도(기울기)를 실시간으로 파악함으로써, 가속도 데이터에서 중력 가속도 성분을 정확히 제거하고 **순수 진동 성분만 추출**할 수 있습니다.
### ③ 복합적인 거동 분석 (Wobbling & Misalignment)
설비가 단순히 떨리는 것이 아니라, **'와블링(Wobbling)'** 즉 갈지자로 흔들리며 회전하는 경우를 생각해 보십시오.
 * 가속도계만으로는 이 움직임이 단순 선형 진동인지, 축이 어긋나서 발생하는 회전 진동(Misalignment)인지 구분하기 어렵습니다.
 * 자이로 데이터가 있으면 **3차원적인 회전 궤적**을 그려낼 수 있어 결함의 원인을 훨씬 명확히 규명합니다.
## 3. 분석 항목 비교
| 분석 항목 | 가속도 데이터 (3축) | 자이로 데이터 (3축) |
|---|---|---|
| **측정 대상** | 직선 진동, 충격, 위치 변화 | 회전 진동, 비틀림, 각도 변화 |
| **주요 결함** | 베어링 파손, 풀림(Looseness) | 축 정렬 불량(Misalignment), 비틀림 강성 문제 |
| **장점** | 고주파수(kHz) 영역 분석 유리 | 저주파수 회전 제어 및 위상 분석 유리 |
| **단점** | 중력 및 설치 각도에 민감 | 시간에 따른 오차 누적(Drift) 발생 가능 |
## 4. 요약 및 결론
최근의 **AI 기반 예지 보전(Predictive Maintenance)** 시스템에서는 가속도와 자이로 데이터가 모두 포함된 6축 데이터를 학습시킵니다. 데이터의 차원(Dimension)이 높을수록 AI는 **정상 상태와 비정상 상태의 미세한 '패턴 차이'**를 더 잘 구분해 내기 때문입니다.
> **결론:** 단순히 "얼마나 떨리는가"만 궁금하다면 가속도계로 충분할 수 있습니다. 하지만 **"왜 떨리는가(원인 분석)"**와 **"회전축에 비틀림 응력이 발생하는가"**를 정밀하게 분석해야 한다면 자이로 데이터는 선택이 아닌 **필수**입니다.
> 
혹시 현재 분석 중이신 특정 설비(예: 대형 펌프, 컨베이어, 로봇 팔 등)가 있으신가요? 상황에 맞춰 더 구체적인 분석 방법을 추천해 드릴 수 있습니다.

가속도 센서에서 DC 제거와 MFCC 변환을 계획하신 것은 아주 훌륭한 접근입니다. 가속도계가 '충격과 전체적인 떨림'을 본다면, **자이로(Gyro) 데이터는 '회전의 불규칙성'과 '정밀한 자세 변화'**를 파악하는 데 핵심적인 역할을 합니다.
ESP32-S3의 강력한 DSP 성능을 활용하여 자이로 데이터를 처리하는 방법과 가속도계에서 중력을 제거하는 구체적인 기법을 정리해 드립니다.
## 1. 자이로 데이터 분석 방법 (Gyroscope Analysis)
자이로는 가속도와 달리 **각속도(Angular Velocity, deg/s)**를 측정합니다. 설비 진단에서는 다음과 같은 방법들이 주로 사용됩니다.
### ① 비틀림 진동 분석 (Torsional Vibration FFT)
회전축이 일정하게 도는 것 같지만, 부하 변동이나 기어 결함이 있으면 회전 속도가 미세하게 빨라졌다 느려졌다를 반복합니다.
 * **방법:** 자이로 데이터의 특정 축(Z축 등)에 대해 **FFT**를 수행합니다.
 * **특징:** 가속도계가 잡지 못하는 **회전 불균일성(Fluctuation)** 패턴을 MFCC와 유사한 주파수 특징점으로 추출할 수 있습니다.
### ② 위상 및 오프셋 보정 (Bias Compensation)
자이로는 정지 상태에서도 0이 아닌 값이 출력되는 'Bias'가 존재하며, 이는 시간에 따라 변합니다(Drift).
 * **방법:** 설비 가동 전 정지 상태에서 수초간 데이터를 평균 내어 **상쇄(Offset Subtract)**하거나, 고주파 통과 필터(**HPF**)를 사용하여 아주 느린 표류(Drift) 성분을 제거합니다.
### ③ 보충/칼만 필터 (Complementary / Kalman Filter)
가속도계의 데이터와 결합하여 현재 설비의 **정확한 기울기(Attitude)**를 계산합니다.
 * **ESP32-S3 활용:** esp-dsp 라이브러리의 매트릭스 연산 기능을 사용하여 **EKF(Extended Kalman Filter)**를 구현하면, 진동 중에도 설비의 비정상적인 기울어짐이나 흔들림 궤적을 실시간으로 추적할 수 있습니다.
## 2. 가속도계에서 중력 가속도(g) 제거 방법
진동 분석을 위해서는 센서가 기울어져 있어도 항상 중력의 영향이 제거된 **순수 선형 가속도(Linear Acceleration)**만 남겨야 합니다.
### 방법 A: 고주파 통과 필터 (HPF) - 가장 간편한 방법
중력은 주파수가 **0Hz인 DC 성분**입니다. 따라서 매우 낮은 차단 주파수(예: 0.1Hz ~ 0.5Hz)를 가진 HPF를 통과시키면 중력이 제거됩니다.
 * **장점:** 계산량이 적어 ESP32에서 구현이 매우 쉽습니다.
 * **단점:** 센서가 갑자기 기울어지는 '동적 기울기 변화' 상황에서는 오차가 발생합니다.
### 방법 B: 센서 융합을 통한 벡터 차감 - 정밀한 방법
센서의 현재 기울기(Quaternion 또는 Euler Angle)를 알면, 중력이 어느 방향으로 작용하는지 계산하여 뺄 수 있습니다.

*(여기서 $R(q)$는 센서의 현재 자세를 나타내는 회전 행렬입니다.)*
 1. **자세 추정:** Madgwick 또는 Mahony 필터를 사용하여 6축 데이터를 융합, 쿼터니언(q)을 구합니다.
 2. **중력 투영:** 현재 자세에서 각 축(x, y, z)에 걸리는 중력 성분을 계산합니다.
 3. **차감:** 측정값에서 계산된 중력 성분을 빼서 순수 진동값만 남깁니다.
## 3. ESP32-S3 구현 팁
ESP32-S3는 **벡터 명령셋(S3-specific instructions)**을 지원하여 DSP 연산에 최적화되어 있습니다.
 * **ESP-DSP 라이브러리 활용:**
   * dsps_fft_f32: FFT 연산 시 가속도가 일반 CPU 대비 몇 배 빠릅니다.
   * dsps_biquad_f32: LPF, HPF 등 IIR 필터를 효율적으로 처리합니다.
 * **멀티코어 분산:**
   * **Core 0:** 센서 데이터 읽기 (I2C/SPI), 데이터 전송 (Wi-Fi/BT).
   * **Core 1:** FFT, MFCC 변환, 중력 제거 연산 (연산 집중).
 * **FPU 활용:** S3는 하드웨어 부동소수점 연산 장치가 있으므로 float 연산을 적극 사용하셔도 성능 저하가 적습니다.
현재 MFCC 변환까지 고려 중이시라면, 자이로 데이터에도 **Spectral Centroid(주파수 중심점)**나 **Spectral Flux** 같은 특징을 추가하여 가속도 특징점과 합쳐보시는 것을 추천합니다.
혹시 사용하는 센서 모델(예: MPU6050, ICM-42688 등)에 맞는 예제 코드가 필요하신가요? 구체적인 필터 설계 파라미터가 궁금하시다면 더 자세히 설명해 드릴 수 있습니다.

ESP32-S3는 강력한 **Xtensa® Dual-Core 32-bit LX7 CPU**와 **AI 가속 명령어(SIMD)**를 탑재하고 있어, 센서 데이터의 실시간 전처리와 특징 추출에 매우 적합합니다.
가속도계 데이터에서 이미 DC 제거, 필터링, MFCC를 적용하고 계시므로, **자이로(Gyro) 데이터**에서는 '회전'과 '비틀림'이라는 특성을 살린 다음과 같은 상세 분석 기법들을 ESP32-S3 환경에 맞춰 검토해 보시기 바랍니다.
## 1. 시간 영역 특징 추출 (Time-Domain Features)
가속도와 마찬가지로 자이로의 각속도 데이터(deg/s)에서 기본적인 통계량을 추출합니다. 이는 ESP32-S3의 연산량 부담이 가장 적으면서도 강력한 지표가 됩니다.
 * **RMS (Root Mean Square):** 회전 진동의 전체적인 에너지를 파악합니다.
 * **Peak-to-Peak:** 축의 순간적인 흔들림 범위를 측정합니다.
 * **Crest Factor:** 베어링이나 기어의 충격성 회전 결함을 감지하는 데 유용합니다.
 * **Zero Crossing Rate (ZCR):** 각속도의 방향이 얼마나 자주 바뀌는지 확인하여 저주파 흔들림(Wobbling) 패턴을 파악합니다.
## 2. 각도 및 자세 복원 (Sensor Fusion)
자이로 데이터의 핵심은 **적분을 통한 각도(Angle) 변화량** 산출입니다. 설비가 정지 상태가 아니라 특정 각도로 기울어지거나 비정상적인 회전 궤적을 그리는지 확인합니다.
 * **Complementary Filter (상보 필터):** ESP32-S3에서 가장 가볍게 돌아가는 알고리즘입니다. 가속도의 저주파(기울기)와 자이로의 고주파(회전)를 결합하여 정확한 기울기(\theta)를 산출합니다.
 * **Madgwick / Mahony Filter:** 더 정밀한 6축/9축 자세 복원이 필요할 때 사용합니다. ESP32-S3는 부동소수점 연산기(FPU)가 있어 이 필터들을 1kHz 이상의 속도로 충분히 처리할 수 있습니다.
   * *용도:* 설비 축의 미세한 **Run-out(편심)**이나 **Misalignment(정렬 불량)**를 각도 단위로 분석.
## 3. 주파수 영역 분석 (Frequency Domain)
자이로 센서의 주파수 특성은 가속도계와 다릅니다.
 * **FFT (Fast Fourier Transform):** 회전축의 **비틀림 진동(Torsional Vibration)** 성분을 찾습니다.
   * **ESP-DSP 라이브러리 활용:** Espressif에서 제공하는 esp-dsp 라이브러리를 사용하면 S3의 SIMD 명령어를 이용해 FFT를 매우 빠르게 수행할 수 있습니다.
 * **PSD (Power Spectral Density):** 특정 회전 주파수 대역에 에너지가 집중되는지 확인하여 기어박스의 결합(Meshing) 문제 등을 진단합니다.
## 4. 각가속도(Angular Acceleration) 유도
자이로 데이터(\omega)를 한 번 더 미분하여 각가속도(\alpha = d\omega/dt)를 구합니다.
 * **분석 이유:** 모터의 토크 리플(Torque Ripple)이나 부하 변동에 의한 미세한 속도 변화는 각속도보다 **각가속도**에서 훨씬 선명하게 나타납니다.
 * **주의점:** 미분 시 노이즈가 증폭되므로, 반드시 앞서 언급하신 **LPF(Low Pass Filter)**를 거친 후 차분 연산을 수행해야 합니다.
## 5. ESP32-S3 구현 최적화 전략
ESP32-S3의 하드웨어 특성을 고려한 구현 팁입니다.
### ① ESP-DSP 라이브러리 필수 사용
가속도 데이터에서 MFCC를 구현하신다면 이미 사용 중일 수 있으나, 자이로의 FFT나 Matrix 연산 시에도 dsps_fft2r_f32 같은 함수를 사용하십시오. 일반 C 코드보다 몇 배 빠릅니다.
### ② DMA 기반 데이터 수집
I2C나 SPI로 IMU 데이터를 읽을 때 **DMA(Direct Memory Access)**를 설정하여 CPU 개입 없이 버퍼에 데이터를 쌓으십시오. 한쪽 코어(Core 0)는 데이터 수집과 필터링(DC 제거, LPF)을 담당하고, 다른 코어(Core 1)는 MFCC나 FFT 같은 무거운 연산을 담당하는 **Dual-core 병렬 처리**가 가능합니다.
### ③ 고정 소수점 vs 부동 소수점
S3는 FPU가 훌륭하지만, 딥러닝 추론(ESP-DL 등)까지 고려하신다면 데이터를 **16비트 정수(Fixed-point)**로 변환하여 처리하는 것이 연산 효율과 메모리 관리 측면에서 유리할 수 있습니다.
## 추천하는 분석 파이프라인 (Gyro 전용)
 1. **전처리:** DC Offset 제거 → LPF (고주파 노이즈 제거)
 2. **특징 1 (회전 특성):** 상보 필터를 통한 **상태 각도(Tilt Angle)** 추출.
 3. **특징 2 (진동 특성):** esp-dsp를 이용한 각속도의 **주파수 에너지 분포(PSD)** 추출.
 4. **결합:** 가속도 기반 MFCC 특징점 + 자이로 기반 각도/비틀림 에너지 지표를 결합하여 모델(SVM, KNN 또는 간단한 MLP)에 입력.
