# BMI270 Driver Design Specification

## 1. 목적

이 문서는 ESP32-S3 기반 IMU Logger에서 **BMI270 드라이버 설계 기준**을 정의한다.

설계 목표:

* 1000Hz 안정 샘플 취득
* FIFO/Interrupt 기반 저지터 구조
* Gyroflow용 데이터 품질 확보
* 자동/수동 calibration 지원
* orientation 및 scale 일관성 확보

---

## 2. 드라이버 책임 범위

BMI270 드라이버는 다음 책임을 가진다.

* BMI270 초기화 및 칩 ID 검증
* SPI 통신 및 레지스터 접근
* FIFO/DRDY 인터럽트 처리 지원
* 샘플링 주파수/범위/필터 설정
* 자이로 bias calibration 적용
* raw frame decode
* 상태/오류 플래그 제공

드라이버는 다음을 직접 담당하지 않는다.

* SD 저장
* 파일 포맷 인코딩
* LED/버튼 UX
* 상위 상태 전이

---

## 3. 초기화 시퀀스

```text
power stable
→ SPI init
→ chip id read
→ soft reset
→ config file load/init sequence
→ accel/gyro range setting
→ ODR/BW setting
→ FIFO setting
→ interrupt mapping
→ self-check
→ ready
```

초기화 실패 시 구분:

* SPI 통신 실패
* chip id mismatch
* config apply 실패
* FIFO init 실패
* interrupt mapping 실패

## 5. 샘플링 모드

### 5.1 권장 모드

1. **High Quality**

   * 1000Hz
   * gyro ±2000dps
   * accel ±16g

2. **Balanced**

   * 400Hz
   * gyro ±1000dps
   * accel ±8g

3. **Low Power Motion**

   * motion detect 위주
   * 저속 또는 event 기반 wake

### 5.2 BMI270 ODR 연계 원칙

* 최종 external log sample rate와 BMI270 internal ODR를 분리해서 생각
* internal ODR는 aliasing과 jitter 측면에서 상위 여유를 둘 수 있음
* Gyroflow 목적의 기본값은 raw integrity 우선

---

## 6. FIFO/인터럽트 설계

### 6.1 권장 구조

* BMI270 FIFO 사용
* watermark interrupt 사용
* 필요 시 data-ready fallback 제공

### 6.2 ISR 원칙

ISR에서는 최소 동작만 수행한다.

```text
ISR
→ event flag set
→ task notify / queue push
→ return
```

ISR에서 금지:

* 긴 SPI read
* dynamic allocation
* complex parsing

### 6.3 Acq Task 처리 흐름

```text
event wait
→ fifo length read
→ burst read
→ raw frame decode
→ timestamp assign
→ sample validate
→ enqueue to logger pipeline
```

### 6.4 FIFO watermark 전략

* 너무 낮으면 interrupt 빈도 증가
* 너무 높으면 latency/jitter 증가
* 1000Hz 기준 수 ms 단위 batch가 적절
* 실측 benchmark 후 보정 필요

---

## 7. timestamp 설계

### 7.1 원칙

* 샘플 timestamp는 monotonic 기준
* RTC wall-clock은 파일 메타용
* FIFO batch 내 개별 sample timestamp를 균등 간격 보간 가능

### 7.2 권장 방식

1. batch read 시점 monotonic tick 확보
2. batch sample count 확인
3. ODR 기준으로 각 sample tick 재구성
4. monotonic 역전 여부 검사

### 7.3 검증 항목

* tick 역전 여부
* batch 간 간격 이상 여부
* sample drop 추정 가능 여부

---

## 8. raw frame decode

드라이버 출력은 가급적 **정규화 전 raw 정수값** 중심으로 유지한다.

출력 구조 예시:

```text
struct Bmi270RawSample {
  uint32_t tick_us;
  int16_t gx;
  int16_t gy;
  int16_t gz;
  int16_t ax;
  int16_t ay;
  int16_t az;
  int16_t temp;
  uint16_t status;
}
```

status bit 예:

* overflow
* clipped
* interpolated_tick
* calibration_applied

---

## 9. calibration 설계

### 9.1 목표

* 정지 상태에서 gyro bias 추정
* bias를 저장하고 이후 세션에 활용

### 9.2 auto calibration

```text
boot/record start
→ stillness detect
→ N samples accumulate
→ mean bias compute
→ threshold validate
→ save/apply
```

### 9.3 stillness 판정

* accel magnitude 변화량
* gyro variance
* 일정 시간 window 유지

### 9.4 실패 정책

* 기존 calib.json 값 사용
* 없으면 uncalibrated flag 유지
* 사용자에게 warning 표시

### 9.5 수동 calibration

* READY 상태에서만 허용
* button 또는 web UI로 실행
* 완료 시 calib.json 저장

---

## 10. orientation 처리 정책

원칙:

* 센서 raw 값은 물리적으로 읽은 순서를 유지
* orientation은 메타데이터 및 export 단계에서 적용 가능하게 설계
* 필요 시 runtime logical mapping 제공

지원 프리셋:

* top_mount
* side_left
* side_right
* inverted
* custom_matrix

---

## 11. motion detect / wake

BMI270 내장 motion 기능을 사용해 sleep 상태에서 wake trigger를 지원한다.

기능:

* no-motion
* any-motion
* threshold based wake

정책:

* sleep 중에는 저전력 감시 모드
* wake 후 full sampling mode 복귀
* false wake 방지를 위한 debounce 필요

---

## 12. 오류/예외 처리

오류 유형:

* SPI timeout
* chip id mismatch
* FIFO overflow
* invalid frame
* impossible sample gap
* repeated interrupt storm

대응:

* transient → retry
* repeated → sensor re-init
* fatal → upper layer notify + recording stop

진단 카운터:

* fifo_overflow_count
* reinit_count
* invalid_frame_count
* timestamp_anomaly_count
