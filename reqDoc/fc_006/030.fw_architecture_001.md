# Firmware Architecture Specification

## 1 목적

ESP32‑S3 기반 IMU Logger 펌웨어의 **모듈 구조 및 책임 정의**.

설계 목표

* 모듈화
* RTOS 기반 병렬 처리
* 유지보수 용이성
* SDMMC 1-bit 저장 경로 반영

---

## 2 전체 구조

```
App Layer
Service Layer
Driver Layer
HAL Layer
```

---

## 3 주요 모듈

### AppController

시스템 전체 초기화

```
init()
run()
shutdown()
```

---

### SystemStateMachine

상태 관리

```
BOOT
READY
RECORDING
USB
WIFI
ERROR
```

---

### ImuManager

BMI270 제어

기능

* init
* read fifo
* calibration

---

### LoggerService

데이터 기록

기능

* buffer 관리
* chunk 생성
* marker 기록

---

### FileManager

SD 파일 관리

기능

* open
* rotate
* finalize
* recovery

---

### StorageManager

저장장치 추상화

기능

* SD mount/unmount
* SD health query
* write/read abstraction
* segment path policy

---

### SdmmcStorageDriver

SDMMC 1-bit 전용 드라이버

기능

* card init
* bus width 1-bit 설정
* file I/O backend 연동
* card detect / reinit
* latency measurement hook

---

## 5 메모리 구조

```
Internal RAM
  └ high speed buffers

PSRAM
  └ secondary buffers

LittleFS
  └ config

SDMMC SD Card
  └ logs
```

---

## 6 데이터 흐름

```
BMI270
 → ImuTask
 → RingBuffer
 → ChunkBuilder
 → SdWriteTask
 → StorageManager
 → SDMMC 1-bit card
```

---

## 7 오류 복구

```
IMU error → restart sensor
SD init/mount error → retry/reinit
write latency spike → buffer headroom absorb
buffer overflow → warning
fatal → stop logging
```
