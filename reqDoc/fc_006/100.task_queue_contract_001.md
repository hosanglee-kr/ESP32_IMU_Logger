# task_queue_contract_001.md

# IMU Logger Task / Queue Contract Specification

---

# 1. 목적

이 문서는 IMU Logger 펌웨어에서 사용하는 **FreeRTOS Task / Queue / Buffer / Event 계약**을 정의한다.

목표

* task 간 책임 경계 명확화
* queue와 ring buffer의 사용 기준 통일
* 실시간 로깅 hot path 보호
* ownership / lifetime / backpressure 정책 표준화

---

# 2. 설계 원칙

## 2.1 기본 원칙

* **고속 샘플 데이터는 queue보다 ring buffer / buffer pool 우선**
* **상태 이벤트 / 버튼 이벤트 / 제어 이벤트는 queue 사용**
* **SD write ownership은 **``** 단일 소유**
* **상태 전이 ownership은 **``** 또는 상태 머신 단일 소유**
* **설정 저장 ownership은 **``** 단일 소유**
* ISR에서는 최소 이벤트만 발행

## 2.2 금지 원칙

* ISR에서 동적 메모리 할당 금지
* ISR에서 긴 SPI/SD I/O 금지
* 다수 task가 같은 file handle 직접 접근 금지
* task 간 raw pointer lifetime 불명확 전달 금지

---

# 3. 전체 task 목록

```text
ImuTask
LogTask
SdWriteTask
UiTask
SystemTask
WebTask
MaintenanceTask
UsbTask (선택적 분리)
```

역할 요약

* `ImuTask` : BMI270 취득
* `LogTask` : chunk 생성 / marker merge
* `SdWriteTask` : SDMMC write / finalize
* `UiTask` : 버튼/LED 처리
* `SystemTask` : 상태 머신 / 상위 제어
* `WebTask` : WiFi / REST API / 파일 조회
* `MaintenanceTask` : stats / benchmark / housekeeping
* `UsbTask` : USB attach/detach / MSC 전환 관리

---

# 4. 권장 우선순위

```text
ImuTask        : highest
LogTask        : high
SdWriteTask    : high
SystemTask     : medium-high
UiTask         : medium
MaintenanceTask: medium-low
WebTask        : low
UsbTask        : medium
```

원칙

* 실시간 취득 > 조립 > 저장 > 제어 > UI > 웹
* WebTask는 로깅 hot path를 방해하지 않도록 가장 낮게 유지 권장

---

# 5. IPC(Inter-Task Communication) 유형

## 5.1 Queue 사용 대상

* 버튼 이벤트
* 상태 전이 요청
* USB attach/detach 알림
* WiFi enter/exit 요청
* 저전압 / 오류 알림
* calibration start/stop 요청

## 5.2 Ring Buffer 사용 대상

* 고속 IMU raw sample stream
* 필요 시 marker/event ingest merge 전 임시 누적

## 5.3 Buffer Pool 사용 대상

* chunk buffer
* write-ready buffer
* recycle buffer

## 5.4 Event Group / Notification 사용 대상

* ISR → ImuTask wake
* flush done 신호
* stop requested 신호
* usb attach state signal

# 6. 주요 queue 정의

# 6.1 SystemEventQueue

목적

* 상태 머신 입력 이벤트 전달

producer

* UiTask
* WebTask
* PowerManager
* UsbTask
* MaintenanceTask
* SdWriteTask

consumer

* SystemTask

payload 예시

```text
struct SystemEvent {
  uint16_t type;
  uint16_t source;
  uint32_t arg0;
  uint32_t arg1;
}
```

대표 이벤트

* RECORD_START_REQ
* RECORD_STOP_REQ
* WIFI_ENTER_REQ
* USB_ATTACH
* LOW_BAT_WARN
* LOW_BAT_CRITICAL
* ERROR_FATAL

---

# 6.2 ButtonEventQueue

목적

* 버튼 드라이버/매니저에서 디코딩된 사용자 입력 전달

producer

* ButtonManager

consumer

* UiTask 또는 SystemTask

payload 예시

```text
struct ButtonEvent {
  uint8_t button_id;
  uint8_t action;
  uint16_t reserved;
  uint32_t tick_ms;
}
```

action 예

* SHORT
* DOUBLE
* LONG

---

# 6.3 MarkerEventQueue

목적

* 사용자 marker / 자동 이벤트 marker 전달

producer

* UiTask
* EventDetectService
* ExternalSync ISR wrapper

consumer

* LogTask

payload 예시

```text
struct MarkerEvent {
  uint32_t tick_us;
  uint16_t marker_type;
  uint16_t flags;
}
```

정책

* queue overflow 시 drop count 기록
* high-priority marker와 low-priority marker 구분 가능

---

# 6.4 WriteReadyQueue

목적

* seal 완료된 chunk buffer를 `SdWriteTask`에 전달

producer

* LogTask

consumer

* SdWriteTask

payload 예시

```text
struct WriteChunkRef {
  uint16_t pool_index;
  uint16_t chunk_type;
  uint32_t size_bytes;
  uint32_t sequence;
}
```

원칙

* 실데이터 복사 대신 buffer pool index/reference 전달 권장
* buffer ownership은 `SdWriteTask`가 recycle 전까지 보유

---

# 6.5 DiagnosticsQueue

목적

* 비실시간 진단 메시지 전달

producer

* 각 task/service

consumer

* MaintenanceTask / DiagnosticsService

정책

* overflow 허용 가능
* hot path에 영향 주면 안 됨

---

# 7. 주요 ring buffer / buffer pool 정의

# 7.1 ImuRawRingBuffer

목적

* ImuTask가 생성한 raw sample stream 저장

producer

* ImuTask

consumer

* LogTask

원칙

* single producer / single consumer 구조 권장
* lock-free 또는 최소 lock
* 내부 RAM 배치 우선

payload 예시

```text
struct RawSample {
  uint32_t tick_us;
  int16_t gx, gy, gz;
  int16_t ax, ay, az;
  int16_t temp;
  uint16_t status;
}
```

---

# 7.2 ChunkBufferPool

목적

* active chunk / write-ready chunk / recycle chunk 관리

구성

```text
FREE
ACTIVE
SEALED
WRITING
RECYCLE_PENDING
```

owner state

* `LogTask` : ACTIVE/SEALED 준비
* `SdWriteTask` : WRITING
* recycle 완료 후 FREE 복귀

---

# 8. Task별 계약

# 8.1 ImuTask 계약

입력

* ISR notify
* state from SystemTask
* current imu config snapshot

출력

* RawSample to ImuRawRingBuffer
* fatal/recoverable event to SystemEventQueue
* health metrics to DiagnosticsQueue

금지

* SD write 직접 수행 금지
* state transition 직접 수행 금지

---

# 8.2 LogTask 계약

입력

* ImuRawRingBuffer
* MarkerEventQueue
* config snapshot

출력

* WriteChunkRef to WriteReadyQueue
* diagnostics to DiagnosticsQueue
* flush/overflow warning to SystemEventQueue

책임

* sample/marker merge
* chunk seal
* checkpoint 삽입 결정

---

# 8.3 SdWriteTask 계약

입력

* WriteReadyQueue
* stop/finalize request
* file state from FileSegmentManager

출력

* write result to SystemEventQueue
* metrics to DiagnosticsQueue
* recycled buffers to pool

단일 소유권

* file handle
* SDMMC write path
* finalize/sync/close

---

# 8.4 UiTask 계약

입력

* ButtonEventQueue
* state snapshot
* system status snapshot

출력

* SystemEventQueue
* MarkerEventQueue
* LED update to LedManager

정책

* 버튼 short/double/long 해석
* short press marker 요청 가능
* recording start/stop 요청은 SystemEventQueue로만 전달

---

# 8.5 SystemTask 계약

입력

* SystemEventQueue
* current state
* guard condition status

출력

* 상태 전이
* 각 task/service control command
* audit/diagnostics event

단일 소유권

* `SystemStateMachine`

---

# 8.6 WebTask 계약

입력

* HTTP/API request
* config snapshot
* file list snapshot

출력

* config change request to SystemEventQueue/ConfigManager
* file query request
* diagnostics read

제약

* recording hot path blocking 금지
* 대용량 파일 다운로드는 상태/버퍼 정책 준수

---

# 8.7 MaintenanceTask 계약

입력

* DiagnosticsQueue
* periodic timer
* stats snapshot

출력

* stats.json update 요청
* benchmark 요청
* recovery scan 요청
* warning event to SystemEventQueue

---

# 9. Config snapshot 계약

원칙

* 다수 task가 config.json 직접 접근 금지
* `ConfigManager`가 validated runtime snapshot 제공
* task는 snapshot read-only 사용
* 변경 적용 시 generation counter 증가 권장

예시

```text
ConfigGeneration = 17
```

정책

* READY 상태에서 snapshot 교체 우선
* RECORDING 중에는 허용된 subset만 반영하거나 다음 session부터 적용

---

# 10. Backpressure 계약

## 10.1 감시 포인트

* ImuRawRingBuffer fill ratio
* WriteReadyQueue depth
* ChunkBufferPool free count
* max write latency

## 10.2 단계별 대응

1. warning

   * diagnostics increment
   * LED/UI warning
2. severe

   * low-priority marker 제한
3. critical

   * optional diagnostics drop
4. fatal

   * controlled stop request

원칙

* raw IMU sample 보존 최우선
* 부가 정보보다 핵심 샘플을 먼저 보호

---

# 11. Stop / Finalize 계약

정상 정지 흐름

```text
SystemTask: RECORD_STOP_REQ 승인
→ ImuTask: new acquisition stop
→ LogTask: active chunk seal
→ SdWriteTask: queue drain + footer write
→ SystemTask: FLUSH_DONE 수신
→ READY 전이
```

원칙

* stop 이후 새 chunk 생성 금지
* finalize 완료 전 USB_MSC 진입 금지

---

# 12. 오류 전파 계약

오류 전파 원칙

* driver/service는 오류를 event/code로 상위에 보고
* 상태 전이는 SystemTask만 결정
* recoverable / fatal 구분하여 보고

예시

```text
IMU FIFO overflow
→ DiagnosticsQueue increment
→ SystemEventQueue warning

SD write fatal
→ SystemEventQueue ERROR_FATAL
→ state machine handles transition
```

# 14. 최종 권고

이 문서의 핵심은 다음이다.

1. **raw sample = ring buffer**
2. **control event = queue**
3. **write ownership = SdWriteTask 단일 소유**
4. **state transition = SystemTask 단일 소유**
5. **config access = validated snapshot 기반 read-only**

이 문서는 이후 `AppEvents.h`, `EventQueue.h`, `RingBuffer.h`, `BufferPool.h`, task entry 구현의 기준 문서로 사용한다.
