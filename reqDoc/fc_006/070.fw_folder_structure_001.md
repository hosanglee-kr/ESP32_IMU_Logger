# fw_folder_structure_001.md
# Firmware Folder Structure and Code Architecture

## 1. 목적

이 문서는 IMU Logger 펌웨어를 실제 구현 가능한 수준으로 분해한 **폴더 구조 / 파일 책임 / 의존 관계**를 정의한다.

핵심 목표

* 모듈 경계 명확화
* 상태 머신/로깅/저장장치 분리
* SDMMC 1-bit 저장 경로 독립화
* 테스트 및 유지보수 용이성 확보

## 3. 폴더별 책임

### /app

* 시스템 시작점
* 상태 머신
* 전체 기능 오케스트레이션

### /core

* 공통 자료형
* CRC/시간/버퍼/에러코드
* 플랫폼 독립 유틸리티

### /config

* LittleFS JSON 로드/저장/검증/마이그레이션

### /drivers

* 하드웨어 직접 접근
* BMI270 / DS3231 / SDMMC / WS2812 / 버튼 / 배터리

### /services

* 비즈니스 로직
* 로깅, 파일 분할, marker, 이벤트 감지, 진단, 전력 관리

### /tasks

* FreeRTOS task entry 및 루프

### /web

* 설정용 웹 UI/API

### /usb

* MSC/CDC 전환

### /logfmt

* binary record/chunk/footer 정의

### /factory

* 양산/서비스 테스트

### /diag

* 진단/감사 로그/헬스 메트릭

## 4. 핵심 의존 관계 원칙

의존 방향

```text
app
 → services
   → drivers
     → hal/esp32 sdk
```

금지 원칙

* driver가 service를 참조하지 않음
* service가 web/usb 상세 구현에 직접 종속되지 않음
* task는 service orchestration만 담당
* state machine 외부에서 임의 상태 전이 금지

## 5. 주요 데이터 흐름 매핑

### 5.1 로깅 경로

```text
Bmi270Driver
→ ImuService
→ ImuTask
→ RingBuffer
→ ChunkBuilder
→ LoggerService
→ SdWriteTask
→ StorageManager
→ SdmmcStorageDriver
```

### 5.2 Marker 경로

```text
ButtonManager / EventDetectService / ExternalSync
→ MarkerService
→ ChunkBuilder
→ LoggerService
```

### 5.3 설정 경로

```text
WebConfigService / Button UI / Factory reset
→ ConfigManager
→ validation/migration
→ active runtime config distribute
```

## 6. 코드 아키텍처 원칙

* hot path에서 dynamic allocation 금지
* task 간 ownership 명확화
* SD write path 단일 소유자: `SdWriteTask`
* config 저장 단일 소유자: `ConfigManager`
* 상태 전이 단일 소유자: `SystemStateMachine`
* marker 삽입은 `MarkerService` 경유만 허용

## 8. 테스트 전략 연결

단위 테스트 우선 대상

* ConfigValidator
* ConfigMigration
* Crc32
* RingBuffer
* ChunkBuilder
* Marker merge logic
* Event detection thresholds

통합 테스트 우선 대상

* BMI270 → logging pipeline
* SDMMC write stall 대응
* segment rotate + recovery
* stop/finalize sequence
* USB MSC 전환 전후 remount

## 9. 최종 권고

이 폴더 구조는 설계 문서 수준이 아니라 **실제 구현 착수용 구조**로 사용하는 것을 목표로 한다.

특히 이번 단계부터는 다음 3가지를 고정하는 것이 좋다.

1. `StorageManager` / `SdmmcStorageDriver` 분리
2. `ChunkBuilder` / `LoggerService` 분리
3. `ConfigManager` 단일 저장 경로 강제
