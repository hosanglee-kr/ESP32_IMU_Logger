# 5. 하드웨어 구성

## 5.1 SD 인터페이스 변경

본 설계의 SD 저장장치 인터페이스는 기존 SPI 기반에서 **ESP32-S3 SDMMC 1-bit 모드**로 변경한다.

변경 목적

* SPI 공유 버스 충돌 감소
* SD 전용 버스 확보
* sustained write 안정성 향상
* 1000Hz 로깅 headroom 확대
* worst-case stall 대응 여유 확보

설계 원칙

* BMI270는 SPI 유지
* SD Card는 SDMMC 1-bit 전용 버스 사용
* 초기 제품 안정화 우선으로 4-bit 대신 1-bit 채택
* 파일시스템/로깅 상위 계층은 인터페이스 추상화 유지
* 향후 보드 revision에서 4-bit 확장 가능 구조로 설계

권장 사항

* SD 배선 길이 최소화
* pull-up 및 신호 무결성 검토
* 카드 detect / power control 가능 시 반영
* hot-insert 정책은 READY 상태 중심으로 제한

# 10. 이벤트 마커(Event Marker)

Gyroflow 후처리 및 테스트 분석을 위해 **로그 내부에 이벤트 마커 레코드 삽입 기능을 지원한다.**

## 10.1 목적

* 테스트 구간 구분
* 특정 이벤트 위치 기록
* 카메라 동기 지원

## 10.2 사용자 입력 기반 Marker

| 이벤트             | 동작        |
| --------------- | --------- |
| 버튼 short press  | 일반 marker |
| 버튼 double press | 중요 marker |
| 버튼 long press   | 구간 marker |

## 10.3 외부 동기 Marker

외부 GPIO pulse 입력 시 marker 생성

예

* 카메라 REC 버튼
* clap sync

로그 레코드

```
MARKER_EVENT

MARKER_SYNC
```


# 11. 자동 이벤트 감지 Marker

센서 데이터를 분석하여 자동으로 이벤트 마커 삽입.

## 11.1 목적

사용자가 직접 입력하지 않아도 주요 이벤트를 자동 태깅.

## 11.2 감지 이벤트

| 이벤트       | 조건                 |
| --------- | ------------------ |
| 충격        | accel threshold 초과 |
| 각속도 급증    | gyro threshold 초과  |
| free‑fall | accel magnitude 감소 |
| 진동 이상     | high variance      |

## 11.3 동작

```
센서 데이터 분석
→ threshold 검사
→ marker record 삽입
```

## 11.4 Marker 유형

```
EVENT_IMPACT
EVENT_HIGH_ROTATION
EVENT_FREE_FALL
EVENT_VIBRATION_ANOMALY
```

## 11.5 설정 가능 항목

* impact threshold
* gyro threshold
* freefall threshold
* detection window
