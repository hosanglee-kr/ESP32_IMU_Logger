# IMU Binary Log Format Specification

## 1 목적

Gyroflow 및 후처리 툴에서 안정적으로 사용할 수 있는 **고속 IMU Binary 로그 포맷 정의**.

설계 목표

* 고속 기록 (1000Hz 이상)
* 파싱 단순성
* 전원 차단 복구 가능
* 확장 가능한 구조

---

## 2 파일 구조

```
[FILE_HEADER]
[META_BLOCK]
[DATA_CHUNK]
[DATA_CHUNK]
[MARKER_CHUNK]
[DATA_CHUNK]
[FOOTER]
```

---

## 3 FILE HEADER

```
struct FileHeader {
  char magic[4];          // "IMUL"
  uint16_t format_ver;
  uint16_t header_size;

  uint32_t fw_version;
  uint32_t device_id;

  uint64_t rtc_start_unix;
  uint64_t monotonic_start_us;

  uint16_t sample_rate;
  uint16_t gyro_range;
  uint16_t accel_range;

  uint16_t orientation;

  uint32_t reserved;
}
```

---

## 4 IMU DATA RECORD

```
struct ImuRecord {
  uint32_t tick_us;

  int16_t gx;
  int16_t gy;
  int16_t gz;

  int16_t ax;
  int16_t ay;
  int16_t az;

  int16_t temperature;
}
```

record size ≈ **16~20 bytes**

---

## 5 MARKER RECORD

```
struct MarkerRecord {
  uint32_t tick_us;
  uint16_t marker_type;
  uint16_t marker_flags;
}
```

marker type

```
1 USER_MARKER
2 IMPORTANT_MARKER
3 SYNC_MARKER
4 EVENT_IMPACT
5 EVENT_ROTATION
6 EVENT_FREEFALL
7 EVENT_VIBRATION
```

---

## 6 DATA CHUNK

Chunk 구조

```
struct ChunkHeader {
  uint16_t type;
  uint16_t record_count;
  uint32_t chunk_size;
  uint32_t crc;
}
```

Chunk types

```
1 IMU_DATA
2 MARKER
3 CHECKPOINT
```

---

## 7 CHECKPOINT RECORD

전원 차단 복구용

```
struct Checkpoint {
  uint64_t file_offset;
  uint32_t last_tick;
  uint32_t crc;
}
```

주기

```
5초 또는 256KB
```

---

## 8 FOOTER

```
struct FileFooter {
  uint32_t total_samples;
  uint32_t total_markers;
  uint32_t file_crc;
}
```

## 9 파싱 흐름

```
read header
→ read chunk
→ decode records
→ handle markers
→ verify CRC
```
